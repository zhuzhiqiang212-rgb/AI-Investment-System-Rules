★ 已锁定的预测与能力边界（董事长 2026-07-21 拍板 · 汇入）

① 体系的目的是「预测为主、PDCA 验证」——不是算账。
董事长原话：「我不一定按你的预测直接交易，只是验证你预测能力提升的过程。如果你只帮我算，我何必建这么大体系呢。这个是我体系的核心价值。」
→ 记分的对象是预测本身，不是收益。预测错但记录完整、事后不改，不扣信任分；事后篡改原预测＝信任击穿。

② 已锁定预测 #001：「100% 看好 AI」
锁定日 2026-07-21，董事长本人下的判断。记分日 2027-07-21。
这是一条方向性长周期预测，不因阶段性回撤改判（配合持仓档案风控口径③甲案）。

③ 成功定义四级 · 以及当前系统真实能力边界
四级：①上涨前发现（最高）→ ②同步确认 → ③事后解释 → ④没看见。
当前系统只能做到 ②同步确认，结构上做不到 ①上涨前发现。
原因：现行各层数据全部是过去或当下（财报／同比／加速度／股价／估值）， 事情反映进财报之前，尺看不见。
这不是缺陷记录，是记分基准线——评价系统进步，要看它从②往①走了多少，不能拿①的标准苛责今天的②，也不许把②包装成①。

④ 三条绝对底线：不虚构来源与数据 ／ 不事后篡改原预测 ／ 不把主观概率冒充历史统计概率。

这份文件是什么、为什么是"尺"

前面①~⑦层每天做判断、打分、攒确定性。那"怎么打分、怎么算确定性、什么算合理"的规则，必须有一把统一的尺——否则换个AI、过段时间，标准就变样、打分就不可比。

本文件就是那把尺。以后所有PDCA复盘，必须照此规则打分，不得随意变。它是"被拿来量对错的准绳"，不是"被验证的料"。

核心：PDCA 分两层验（董事长明确要求）

复盘不只看"判断结果准不准"，还要看"我们判断的方法、定的规则本身合不合理"。规则本身也是可错的、要被持续迭代的对象（呼应体系总则第五条"总则也在被检验，不是不可错的教条"）。

第一层 · 结果层：这个判断（如"国家战略=AI"）后来被事件证明对没对？
第二层 · 规则方法层：我们怎么形成这个判断的方法、给它定的打分规则本身，合不合理？要不要改？

第一层 · 结果层记分规则

每日打分：今天发生的事件，对每个判断投一票——

今天的事件对这个判断打分
支持（事件证明判断成立）+1
中性（无相关新事件）0
证伪（事件推翻判断）−1

确定性升降级（累计分决定把握度）：

连续被支持 → 升级：弱 → 中 → 高
连续被打脸 → 降级：高 → 中 → 弱 → 证伪（证伪则切备用逻辑）
例·"总闸=美国优先"：7-02立(中/+0) → 缩表数据支持(中/+1) → 定向放美元支持(中偏高/+2) → …连续5天支持 → 升级到"高" → 这时重仓AI供应链才有底气。

第二层 · 规则方法层自省规则（董事长强调·核心）

定期回看"我们判断的方法、打分的规则"本身合不合理，防止"用一套有偏的方法，越来越熟练地打错分还不自知"。

自省项问什么何时查
判断形成方法这一环（如国家战略）我们用哪些指标、什么逻辑推出判断？抓对关键了吗？月/季
打分标准我们定的"什么算支持/证伪"，标准合理吗？会不会一直误判？月/季
确定性阈值"连续几天支持才升级"这个阈值，太松还是太紧？季/年
规则迭代发现不合理 → 改规则 → 记录为什么改（规则也进版本管理）随时发现随时提

铁律：规则不是定了就永不变。规则本身也进PDCA、也被迭代。改规则要记录理由、董事长确认。

决策质量分（最终落点）

把各判断的确定性综合，回答"我现在下手有多少底气"，直接影响①~⑦层的收/放口径：

· 相关判断多为"高" → 决策质量高 → 敢按方向下手、机会池放开
· 多为"中" → 决策质量中 → 收着点、只守核心（今天2026-07-02即此）
· 有判断被"证伪" → 决策质量低 → 停用相关逻辑、切备用

多尺度复盘（各尺度都跑这套规则）

尺度看什么
日今天各判断打了几分、持仓动作对没对
周本周累计分、哪些判断在夯实/动摇、决策命中率
月确定性升降级的判断、+第二层：判断方法/规则合理吗
季证据链骨架要不要调、+确定性阈值合理吗
年全年判断准确率、决策质量趋势、+总则/规则要不要迭代

保持动态（不违反总则第六条防静态化）

本准绳定的是规则框架（怎么加减分、怎么升降级）——框架固定。但每天的分、确定性、决策质量，全是当天事件现算，不写死任何结论。规则本身也随第二层自省而迭代。既动态、又有稳定的尺。

KPI · 记分要奔的硬指标（董事长定 · 2026-07-08 绑入）

指标目标怎么记
命中率（判断准确率）55% → 60% → 70%（随时间往上爬）结果层打分累计；宏观 / 个股分开记
"高把握还打脸"率< 10%标"高"确定性的判断里被证伪的占比；超10%触发第二层规则自省
富途账户（落地 C）年化 +100% 起、上不封顶围绕此数做进取 / 收敛，作决策质量的落地检验

大脑(4账户)记"判得准不准"(命中率/打脸率)，落地(富途)记"赚没赚到"(年化)；两者都进复盘。KPI 达不达标，是规则是否要迭代的信号。

决策卡设计原则（铁律 · 2026-07-08 绑入 · 治"精简过头看不懂"）

  1. 每张卡一个明确动作：买 / 卖 / 守，配人话理由。
  2. 一个价不能既买又卖：禁止"低吸线＝止损线"这种自相矛盾的价位。
  3. 对比、分析、图文并茂、说透：机会池 / 持仓 / 决策要摆对比、讲清楚。
  4. 通俗易懂、不许精简到看不懂：术语当场用人话解释。
  5. 左栏现算 · 动态（改数据就变）；只锚定义、不锚死名单；带当日指纹。

此 5 条为决策输出（持仓 / 机会池 / 今日决策卡）的验收铁律，绑所有产出岗位；违反即打回。
