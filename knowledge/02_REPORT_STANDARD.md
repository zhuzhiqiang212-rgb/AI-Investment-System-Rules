## 一、日报定位（唯一主输出）

### 核心定位

日报是每日投研决策的入口，不是数据展示，不是研报。

日报回答:
  今天做什么
  今天不能做什么
  未来3天怎么办（简版，含条件触发）
  哪些仓位危险
  哪些仓位值得关注
  是否需要生成研报

日报不回答:
  为什么这么做（研报职责）
  长期怎么看（研报职责）
  事件完整推演（研报职责）
  个股深度逻辑（研报职责）

### 文件规范

文件名: 00_今日日报.pdf（固定，每日覆盖）
日期归档: AI投研日报_YYYY-MM-DD.pdf
输出路径: C:\Users\zhu20\OneDrive\桌面\股票分析与研究\
覆盖规则: 同日期自动覆盖

### 日报硬性长度约束

最多3页。如果超过3页，优先检查哪些内容应移入研报。
不以压缩信息为代价，而是以分流到研报为手段。

## 二、日报必须包含的8个模块

### 模块1: 今日决策摘要（60秒版）
今天做什么 / 最不能做什么 / 最大机会 / 最大风险 / 今天关注

### 模块2: 市场数据
VIX / 10Y / SPX / BTC，含OK/DATA_GAP状态

### 模块3: 今日结论（含子周期）
总周期 + 置信度 + 判断依据
AI股/加密/日股/A股四个子周期
第1层动作（按账户输出，禁止占位符）
第2层最不能做（至少2条，含当日具体数据）

### 模块4: 未来3日条件触发计划
每个关键事件有至少2个情景分支
每个分支有具体行动（非"等待确认"）
事件情景只写简版（详细推演在研报）

### 模块5: 持仓分层速览
核心仓 / 高Beta仓 / 防守仓 / 观察仓 / 问题仓

### 模块6: 资金轮动主线（5行以内）
当前主线 / 备选主线 / 观察中 / 退潮板块 / 现金状态

### 模块7: 四账户今日动作
富途 / SBI / IBKR / BF 各自今日关注 + 动作边界 + 禁止

### 模块8: 研报触发卡（触发时必须输出）
格式: 任务卡，不展开研报正文
触发后Claude生成研报，研报结论反哺日报

## 三、研报定位

### 核心定位

研报是V6投研体系的深度层，负责机会发现和进攻判断。

研报回答:
  为什么这么做
  长期怎么看
  下一主线在哪里
  什么时候进攻
  事件完整推演（A/B/C三情景完整版）
  个股深度分析

研报不替代:
  每日决策入口（日报职责）
  今天做什么的简洁答案（日报职责）

### 研报类型（详见RESEARCH_ENGINE_STANDARD.md）

| 类型 | 触发 | 频率 |
|------|------|------|
| 事件专题研报 | ★★★★以上事件在3日内 | 事件驱动 |
| 个股深度研报 | 浮盈>50%或浮亏<-30% | 事件驱动 |
| 进攻机会研报 | 周期升级为BULL A级 | 事件驱动 |
| 趋势轮动研报 | 每周五 / 主线切换 | 周更+事件驱动 |
| 月度归因报告 | 每月末 | 月更 |

## 四、日报与研报联动规则

### 联动链路

日报输出研报触发卡（任务卡）
  ↓
用户确认或自动触发
  ↓
Claude生成对应研报
  ↓
研报结论以"见[研报名称]"一行引用写回日报

### 禁止的联动方式

禁止: 研报正文内容塞进日报
禁止: 日报因承担研报职责无限变长
禁止: 日报代替研报输出深度分析
禁止: 研报触发卡触发后不生成研报（停留在触发状态）

## 五、日报质量标准（V6.2版，11项）

1. 持仓无重复（每个code唯一）
2. 今日决策摘要存在且layer1非占位符
3. 子周期判断出现（AI股/加密/日股/A股四项）
4. layer1按账户输出（富途/SBI/IBKR/BF各自一行）
5. 未来3日计划每个事件有至少2个情景分支
6. 每个情景分支有具体行动
7. 持仓分层出现（5类）
8. 资金轮动主线出现（5行以内）
9. 所有条件触发动作含"执行前人工确认"声明
10. 研报触发卡格式正确（触发时必须输出任务卡）
11. 日报页数不超过3页

任意FAIL → 不生成PDF，输出失败原因
