## 一、系统定位

AI投研总控台 V6 是一个**攻守兼备**的投研决策辅助系统。

最终目标: 四账户年化收益率 40%+（挑战目标，不是上限）
          趋势强时允许60-80%，趋势极强时允许100%+
交付物: 每天一份《00_今日日报.pdf》+ 按需生成专题研报
核心边界:
  系统提供决策辅助，不自动执行
  账户下单和出入金只能由用户本人操作
  禁止连接券商下单接口
  禁止存储账户密码或API密钥

## 二、双引擎架构（V6.2核心变化）

### 第一引擎：日报引擎（防守 + 决策入口）

职责: 每日自动生成，回答"今天怎么办"
输出: 00_今日日报.pdf
核心能力:
  周期判断（总周期+子周期）
  持仓分层（核心/高Beta/防守/观察/问题）
  四账户各自动作
  资金轮动主线（规则自动判断）
  未来3日条件触发计划
  研报触发卡（发现机会时自动触发）

### 第二引擎：研报引擎（进攻 + 机会发现）

职责: 事件驱动或周更，回答"为什么/长期怎么看/机会在哪"
输出: 各类专题研报PDF（见RESEARCH_ENGINE_STANDARD.md）
核心能力:
  事件完整推演（BOJ/FOMC/三巫日等）
  个股深度分析（止盈止损逻辑）
  趋势轮动判断（资金从哪里来/流向哪里）
  进攻机会识别（B级埋伏/A级主动出击）
  月度归因复盘

**防守与进攻同等重要。系统必须同时具备两种能力。**

## 三、模块职责

数据层: daily_data_fetch.py + futu_position_fetch.py
  → Yahoo Finance / Binance / 富途OpenD

分析层: auto_briefing.py
  → 总周期 + 子周期 + 置信度 + 信号等级

决策层: auto_briefing.py
  → 三层结论 + 四账户动作 + 持仓分层 + 轮动主线

输出层:
  主输出（用户查看）: 00_今日日报.pdf
  研报输出（按需）: 各类专题研报PDF
  内部文件（不向用户主动展示）: latest_*.md / 方向卡 / 执行卡 / 风险卡

治理层: governance_runtime.py + skill_gate.py

研报层: Claude（生成）+ ChatGPT V6（审批）
  → 事件专题 / 个股深度 / 趋势轮动 / 进攻机会 / 月度归因

归因层: trade_record_log_v1.md + monthly_return_tracking_v1.md

## 四、职责边界（正式版）

### Claude 负责
  研究 / 规划 / 方案设计 / 数据整理 / 搜索与调研
  文件生成 / 报告生产 / 研报生成 / 任务拆解
  所有建议必须标注"建议方案"，不得直接视为最终制度

### ChatGPT V6 负责
  审批 / 验收 / 最终架构裁决 / 最终治理裁决
  制度解释 / 冲突检测 / 研报方向审批
  拥有 V6 最终裁决权

### Codex 负责
  程序实现 / 自动化 / 文件写入 / 脚本运行
  禁止: 自行发起任务 / 自验 / 生成研报正文 / 修改制度

### 用户 负责
  账户操作 / 制度批准 / 最终确认
  不可被代替

## 五、用户不需要每天重复提醒的规则

以下规则已固化在制度文件中，用户不需要每次提醒：

1. 40%是挑战目标不是上限，趋势强时允许更高收益
2. 系统必须同时具备防守和进攻能力
3. 研报触发后Claude必须生成，不能停留在触发卡状态
4. 日报最多3页，超出部分移入研报
5. 所有进攻建议须含具体账户、标的、条件
6. 精确价格只来自规则库/用户确认/真实数据

## 六、Governance Override Rule

任何新指令影响制度前，必须提示用户选A/B/C：
  A. 临时执行一次
  B. 永久修改制度
  C. 取消执行

未获得用户确认前，禁止直接修改制度。

---

## 七、User-First Rule

用户只负责三件事：打开日报 / 查看内容 / 做投资决策
禁止要求用户运行脚本/打开命令行/手动复制文件

---

## 八、用户界面语言规范

所有面向用户内容必须使用简体中文。
方向卡→方向卡 / 执行卡→执行卡 / 风险卡→风险卡
系统内部允许英文，用户界面统一简体中文。

---

## 九、新模块继承规则

任何新增模块必须：
  继承系统定位（攻守兼备）
  governance_runtime前置检查
  不新增下单接口 / 不存储密码
  输出符合日报或研报标准
  下一步说明用中文
