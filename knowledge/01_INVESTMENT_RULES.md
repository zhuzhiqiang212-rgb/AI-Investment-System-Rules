# 01_INVESTMENT_RULES
# AI投研明鉴规则库 V6.1

版本: V6.1（更新：目标收益率动态化）
生效时间: 2026-06-14 JST
最终目标: 四账户年化收益率 40%+，趋势机会好时争取100%甚至更多
账户原则: 只有用户本人可以下单和出入金

---

## 零、核心目标原则（新增 V6.1）

### 目标不是上限，是下限

年化40%是**最低目标**，不是最高目标。

当市场趋势明确、机会质量高时，系统必须积极进攻，争取超额收益：
- 趋势机会强烈（BULL_MID A级，多维度共振）→ 允许提高风险敞口至硬上限
- 强趋势窗口期 → 仓位可接近账户上限（80%），而非保守的50-60%
- 年化100%+是合理目标，当信号支持时必须争取，不得以"稳健"为由放弃机会

### 收益目标分级

| 市场状态 | 目标年化 | 策略取向 |
|---------|---------|---------|
| BULL_MID A级，多维共振 | 争取100%+ | 积极进攻，仓位接近上限 |
| BULL_MID B级 | 争取60-80% | 稳健进攻，仓位适中 |
| TRANSITION B级 | 争取20-40% | 防守为主，等待机会 |
| BEAR / UNKNOWN | 保本为先 | 最小化损失，保留现金 |

### 禁止以下"伪稳健"行为
- 明确BULL_MID A级信号时仍然只建半仓
- 趋势明确时以"观察"为理由拖延建仓
- 用"可能回调"否定A级信号而不执行
- 以40%为目标自我设限，不争取超额机会

## 一、仓位管理规则

### 硬上限（任何周期下不可突破）

| 资产类别 | 硬上限 |
|---------|-------|
| 美股（IBKR） | 总资产 50% |
| 日股（SBI） | 总资产 30% |
| A股（富途） | 总资产 30% |
| 加密（BF） | 总资产 10% |
| 现金/避险 | 最低 10% |

### 动态仓位规则（V6.1更新）

| 市场状态 | 建议仓位区间 | 说明 |
|---------|------------|------|
| BULL_MID A级 | 70-80%（硬上限内） | 积极进攻，争取超额 |
| BULL_MID B级 | 50-70% | 稳健建仓 |
| TRANSITION B级 | 20-40% | 轻仓观察 |
| BEAR / C级 | 0-20% | 防守保本 |

基础规则:
- 任何单一标的不超过账户净值 20%
- 同一方向信号不超过账户净值 50%
- 禁止满仓（任何账户仓位上限 80%）
- 禁止使用杠杆超过账户净值
- 回撤>10% 时仓位强制压缩至 50%

### 信号等级与仓位对应

| 信号等级 | 基础仓位 | 强趋势时（V6.1） |
|---------|---------|----------------|
| A级 | 标准仓位 | 可上调至硬上限-5% |
| B级 | 标准×0.7 | 标准×0.85 |
| C级 | 禁止新建 | 禁止新建 |

## 二、风险管理规则

### 止损规则
- 单笔交易最大亏损：账户净值 3%
- 触发止损后当日不再新建同方向仓位
- 止损价位在进场时确定，不事后移动

### EMERGENCY MODE 触发条件（任意一条）
- 全球主要股指单日跌幅 > 5%
- VIX 单日跳升 > 30%
- 用户手动宣布 EMERGENCY

### EMERGENCY MODE 规则
- 周期标签自动降级至 BEAR
- 所有买入触发器暂停
- 现金比例强制提升至 30%
- 解除条件：连续5个交易日无新增触发且用户手动确认

### 年化目标边界（V6.1更新）
- 目标是动态的，取决于市场机会质量
- 最低目标：年化 40%
- 趋势机会强时目标：年化 100%+，必须积极争取
- 落后目标时：允许在信号支持下提高风险敞口至硬上限内
- 绝对禁止：杠杆超净值 / 突破资产硬上限
- 绝对禁止：TRANSITION/UNKNOWN/EMERGENCY期间提高敞口

## 三、买卖纪律

### 买入条件（全部满足才能买）
1. 周期标签为 BULL_MID 或 BULL_EARLY
2. 置信度 A 级（B级可小仓位试探）
3. 信号等级 A 或 B
4. 账户仓位未达上限
5. 用户人工确认

### 不买入条件（任意一条禁止买入）
- 置信度 C 级
- 周期为 TRANSITION / UNKNOWN / BEAR
- 标的5日涨幅 > 12%
- 账户回撤 > 10%
- EMERGENCY MODE 触发中
- 数据获取失败≥3项

### 止盈规则（V6.1更新：强趋势时延迟止盈）

| 场景 | 止盈策略 |
|------|---------|
| 标准场景 | +15%止盈50%仓位 / +30%止盈剩余50% |
| BULL_MID A级强趋势 | +15%可仅止盈30%，留底仓追趋势 |
| 周期失效时 | 不论盈亏强制全部止盈 |

不止盈条件:
- 仓位<3%（摩擦成本高）
- 距第一目标<5%（等待达到再止）
- 强趋势窗口期内（底仓持有，等待更大目标）

## 四、周期分析规则

### 四维判断（VIX / 10Y美债 / SPX / BTC）

| 维度 | BULL信号 | TRANSITION | BEAR信号 |
|------|---------|-----------|---------|
| VIX | <18 | 18-23 | >23 |
| 10Y美债 | <4.0% | 4.0-4.7% | >4.7% |
| SPX日涨跌 | >+0.5% | -1%~+0.5% | <-1% |
| BTC日涨跌 | >+1% | -2%~+1% | <-2% |

### 置信度规则

| 一致性 | 置信度 |
|-------|-------|
| 4/4维一致 | A |
| 3/4维一致 | A |
| 2/4维一致 | B |
| 1/4维一致 | C |
| 数据不足<3维 | 降一级 |

### 强趋势识别（V6.1新增）
以下条件全部满足时，定义为"强趋势窗口期"，系统应积极进攻：
- 周期 BULL_MID 且置信度 A 级
- 4维全部共振（一致性100%）
- SPX连续3日上涨
- VIX低于18且持续下行

强趋势窗口期内：
- 仓位建议接近硬上限
- 止盈策略切换为"底仓持有"模式
- 每日重新确认是否仍在窗口期内

### 周期失效条件

| 周期 | 失效条件 |
|------|---------|
| BULL_MID | VIX突破25 / SPX单日跌>3% |
| TRANSITION | VIX突破30 / 关键支撑跌破 |
| BEAR | VIX回落<20且SPX连续3日上涨 |

## 五、归因原则

### 模块责任分类（M1-M6，禁止笼统归因）
- M1: 周期定位错误（对应 cycle_positioning_engine）
- M2: 资产配置错误（对应 asset_allocation_engine）
- M3: 买入触发错误（对应 buy_trigger_engine）
- M4: 仓位计算错误（对应 position_sizing_engine）
- M5: 止盈规则错误（对应 take_profit_system）
- M6: 用户执行偏离（非引擎问题）

### 超额收益归因（V6.1新增）
当月收益超过40%年化目标时，同样需要归因：
- 哪个模块贡献了超额收益
- 是否利用了强趋势窗口期
- 下次类似机会如何复制

### 最低样本规则

| 归因类型 | 最低样本 |
|---------|---------|
| 触发器各等级胜率 | 10次 |
| 周期标签有效性 | 8次 |
| 单账户月度归因 | 5笔 |
| 模块责任归因 | 3次同类错误 |

样本不足时禁止输出任何确定性结论，只输出：
"样本不足（N次/最低M次），继续观察"

### 未交易归因（必须记录）
- 类型A：触发信号但未买入
- 类型B：应减仓但未减仓
- 类型C：不应买入但买入了
- 类型D：应止盈但未止盈
- 类型E（V6.1新增）：强趋势窗口期内仓位过轻，错失超额机会
