## 零、机会发现优先原则（V1.1新增，最高优先级）

研报不仅负责风险分析。
研报必须同等回答机会与风险。

研报必须回答以下问题（不得只回答风险）:

  哪里有机会
  为什么是机会
  什么时候出手
  什么时候放弃
  主线是什么
  备选主线是什么
  资金从哪里流向哪里
  当前是防守期还是进攻期
  是否属于强趋势窗口期
  是否属于A级主动出击机会
  是否属于B级埋伏机会

研报三部分必须同时存在:
  第一部分: 风险分析（不可省略）
  第二部分: 机会判断（不可省略，不得弱化）
  第三部分: 行动计划（不可省略，必须具体到账户和标的）

禁止:
  只写风险不写机会
  只写"观察等待"不给出进攻时机判断
  机会分析比风险分析篇幅少

## 一、研报系统定位

研报系统是V6投研体系的深度层。
日报是决策入口，研报是机会发现和进攻判断的数据来源。

研报负责回答:
  为什么这么做
  长期怎么看
  下一主线在哪里
  什么时候进攻
  事件完整推演
  个股深度逻辑
  哪里有机会（与风险同等重要）

研报不负责:
  今天做什么（日报职责）
  未来3天简版动作（日报职责）

## 二、研报类型定义

### 类型1: 事件专题研报
触发条件: 事件日历中影响★★★★以上事件在未来3日内
命名规则: 事件专题_[事件名称]_YYYY-MM-DD.pdf
示例: 事件专题_BOJ议息_2026-06-16.pdf
生成者: Claude
更新频率: 事件驱动

内容结构（三部分必须同时存在）:
  第一部分: 事件背景（市场预期/历史先例/当前相关数据）
  第二部分: 情景推演完整版（A/B/C三个情景，每个含详细逻辑）
    - 各情景对各账户持仓的具体影响
    - 各情景下的机会判断（此次事件是否创造进攻机会）
    - 各情景下的风险边界
  第三部分: 行动计划（具体到每个账户每个持仓）
    - 当前是防守期/观望期/进攻期的判断
    - 是否存在A级或B级机会
    - 研报结论（回写日报的简版动作依据）

### 类型2: 个股深度研报
触发条件:
  任一持仓浮盈 > +50%（止盈时机研究）
  任一持仓浮亏 < -30%（止损复盘）
  用户手动请求
命名规则: 个股研究_[标的代码]_[主题]_YYYY-MM-DD.pdf
生成者: Claude
更新频率: 事件驱动

内容结构（三部分）:
  第一部分: 当前状态（持仓/浮盈亏/周期匹配度）
  第二部分: 机会与风险分析
    - 若浮亏：是否仍有持有价值/止损逻辑/反转机会
    - 若浮盈：止盈时机/是否继续持有/加仓机会
    - 技术结构（支撑压力）
    - 基本面简析
  第三部分: 条件触发行动计划（若A→做Y / 若B→做Z）

### 类型3: 趋势轮动研报
触发条件: 每周五手动生成 / 子周期出现独立强势信号
命名规则: 趋势轮动_YYYY-WXX.pdf
生成者: Claude
更新频率: 周更 + 事件驱动

内容结构（必须回答机会方向）:
  本周资金流向（从哪里来/流向哪里）
  当前主线与备选主线
  下一轮可能的进攻方向（不可省略）
  各子周期状态（AI股/加密/日股/A股）
  是否存在B级埋伏机会
  下周机会预判

### 类型4: 进攻机会研报
触发条件: 总周期升级为BULL_MID A级 / 强趋势窗口期识别
命名规则: 进攻机会_[标的或板块]_YYYY-MM-DD.pdf
生成者: Claude
更新频率: 事件驱动

内容结构:
  为什么现在是进攻窗口
  首选标的（账户+标的+原因）
  仓位建议（结合position_sizing_engine规则）
  风险边界（止损条件+失效条件）
  何时放弃（退出机制）

### 类型5: 月度归因报告
触发条件: 每月末手动生成
命名规则: 月度归因_YYYY-MM.pdf
生成者: Claude + 用户填写trade_record_log
更新频率: 月更

内容结构:
  实际收益率 vs 挑战目标（40%+）
  M1-M6六模块归因
  本月错过的机会（未交易事件A/B/C/D/E/F类）
  下月进攻机会预判（不只做复盘，还要展望）

## 三、事件驱动触发规则

自动触发条件（由auto_briefing.py检测，日报触发卡输出）:

| 触发条件 | 生成研报类型 | 优先级 |
|---------|------------|--------|
| 事件日历★★★★以上事件在未来3日内 | 事件专题研报 | P0 |
| 任一持仓浮亏 < -30% | 个股深度研报（含机会判断）| P0 |
| 总周期从TRANSITION升级为BULL_MID A级 | 进攻机会研报 | P0 |
| 任一持仓浮盈 > +50% | 个股深度研报（止盈+续持判断）| P1 |
| 子周期出现独立强势信号 | 趋势轮动研报 | P1 |
| 连续3日layer1为"观察" | 机会扫描研报（主动寻找机会）| P2 |
| 每周五 | 趋势轮动周报 | P1 |
| 每月末 | 月度归因报告 | P1 |

手动触发:
  用户说"帮我研究[标的]" → 个股深度研报
  发现重大资金轮动信号 → 趋势轮动研报
  黑天鹅/重大政策事件 → 紧急事件研报

## 四、研报与日报联动规则

联动原则: 日报触发研报，研报结论（含机会判断）支撑日报动作建议

联动链路:
  日报输出研报触发卡（任务卡格式，不展开正文）
    ↓
  Claude生成对应研报（三部分：风险+机会+行动）
    ↓
  研报结论写回日报对应账户动作区
  （格式: "见[研报名称]" 一行引用）
    ↓
  日报保持3页以内

禁止:
  研报正文内容塞进日报
  日报因承担研报职责无限变长
  研报触发卡触发后不生成研报（停留在触发状态）
  研报只分析风险不分析机会
