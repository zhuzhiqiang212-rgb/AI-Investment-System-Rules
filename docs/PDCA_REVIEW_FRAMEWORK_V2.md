# PDCA 复盘验证与改进闭环 V2

## 一、四段闭环
Check：逐条把昨日判断打成对、错、待观察，并记录今日验证数据。

归因：定位哪个环、哪个口径、哪个证据源出错，不把责任写成市场随机波动。

Act：写具体改进方案，只能写某环阈值、某口径、某数据源、某脚本字段，不写空话。

回填：把改进写回证据链骨架或求证表口径，留痕为第X次迭代·因某复盘。

## 二、对错判定口径
决策对：昨天守、不追、低吸等动作被今日走势印证。

决策错：今日走势直接打脸昨日动作或约束。

待观察：一天看不清，需要更长窗口。

## 三、分层复盘
今日复盘：每日开机先验昨天决策。

周复盘：周五执行，验证一周方向稳定性。

月复盘：月末执行，验证战略主线是否持续有效。

季复盘：季末执行，验证总闸大判断与证据链骨架。

年复盘：年末执行，验证体系收益与主逻辑迭代。

## 四、铁律
复盘必须基于存档的历史求证表与决策，即 data/evidence_chain/ 与 data/decisions/。

不改历史，不事后诸葛，不写空泛改进。

归因和改进方案由人填写，脚本只生成机械比对和待填位。
