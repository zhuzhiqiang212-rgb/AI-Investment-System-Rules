# 估值模型库 V1

【总则对照】本文件对照总则六条：第一条→估值从世界观证据链下游的机会池与持仓受检对象进入，不从持仓反推理由；第二至五条→本文件只登记模型类型，不改证据链骨架、每日求证、下游驱动与复盘；第六条防静态化→自评：这里只锚模型类型与所需输入，不锁死具体标的，每日由分派器按当日标的动态挂模型。

## 一、定位
本库是估值第二道关的模型类型登记处。它只定义“什么类型标的用什么估值框架”，不写死“哪些股票必须估值”，也不把估值结论固定为静态答案。

## 二、模型类型

### 1. 成长情景PE
- 适用哪类标的：成长科技股，如微软、英伟达类；标准是业务高增长、利润可前瞻、市场主要用成长倍数定价。
- 所需财务输入：前瞻 EPS、收入或利润增速、毛利率趋势、资本开支压力、熊/基准/牛三档 PE 假设。
- 估值方法：熊/基准/牛三档 PE × 前瞻 EPS，形成三档价格。
- 输出便宜合理贵三带：低于熊档或合理区下沿为便宜；接近基准档为合理；高于牛档或历史高分位为贵。
- 可迭代：是。本模型锚定类型，不锁死具体标的。

### 2. NAV折价
- 适用哪类标的：控股投资公司，如软银类；标准是公司价值主要由持有资产组合与折价率决定。
- 所需财务输入：每股 NAV、核心持仓估值、现金与负债、LTV、历史折价带。
- 估值方法：NAV ×（1 - 折价率），按熊/基准/牛三档折价率形成价格带。
- 输出便宜合理贵三带：折价明显宽于历史中枢为便宜；接近历史中枢为合理；折价明显收窄或溢价为贵。
- 可迭代：是。本模型锚定类型，不锁死具体标的。

### 3. 周期PE
- 适用哪类标的：周期股与设备股，如半导体设备、资本开支敏感公司。
- 所需财务输入：前瞻 EPS、周期位置、订单与资本开支趋势、正常化 PE、峰值盈利警示。
- 估值方法：用正常化 PE 与不同周期阶段 EPS 交叉估值，重点标注峰值盈利高 PE 双杀风险。
- 输出便宜合理贵三带：正常化盈利下低倍数为便宜；接近周期中枢为合理；峰值盈利叠加高 PE 为贵。
- 可迭代：是。本模型锚定类型，不锁死具体标的。

### 4. 医药管线DCF
- 适用哪类标的：医药股，如第一三共类；标准是价值主要由核心管线、专利期与现金流决定。
- 所需财务输入：核心管线、峰值销售、成功率、现金流、专利期、折现率。
- 估值方法：DCF 或峰值销售折现，按管线成功概率与专利期形成情景价格。
- 输出便宜合理贵三带：管线折现后明显高于市值为便宜；接近风险调整价值为合理；充分透支管线成功为贵。
- 可迭代：是。本模型锚定类型，不锁死具体标的。

### 5. 保险内含价值EV
- 适用哪类标的：保险股，如东京海上类；标准是价值主要由内含价值、新业务价值与利率环境决定。
- 所需财务输入：EV、新业务价值、PEV 倍数、承保利润、投资收益、利率敏感度。
- 估值方法：EV × PEV 倍数，并结合新业务价值与利率变化做情景调整。
- 输出便宜合理贵三带：低于历史 PEV 中枢且基本面稳定为便宜；接近中枢为合理；明显高于中枢且利率顺风减弱为贵。
- 可迭代：是。本模型锚定类型，不锁死具体标的。

### 6. 加密mNAV
- 适用哪类标的：加密相关标的，如持币公司、交易所与稳定币相关公司。
- 所需财务输入：持币量或净资产、币价、mNAV 倍数、交易收入、融资与稀释风险、监管状态。
- 估值方法：持币或净资产 × mNAV 倍数，交易所类另看交易收入与盈利弹性。
- 输出便宜合理贵三带：mNAV 低且资产质量可验证为便宜；接近历史区间为合理；mNAV 高且融资稀释加剧为贵。
- 可迭代：是。本模型锚定类型，不锁死具体标的。

## 三、执行边界
分派器只能做“判类型加挂模型”，不能自动填写估值数字。真估值必须由理解岗基于真实财务输入逐只完成，缺输入则标待补。
