# 文件一：00_SYSTEM_MANUAL.md
# AI投研总控台 V6 系统手册

版本: V6.0 | 生效时间: 2026-06-14 JST

## 一、系统定位

AI投研总控台 V6 是一个有边界的决策辅助系统。

最终目标: 四账户年化收益率 40%+，趋势机会好时争取100%甚至更多
交付物: 每天一份决策前判断简报
核心边界:
  - 系统只提供决策辅助，不自动执行
  - 账户下单和出入金只能由用户本人操作
  - 禁止连接券商下单接口
  - 禁止存储账户密码或API密钥

## 二、模块职责

数据层: daily_data_fetch.py → Yahoo Finance（VIX/SPX/10Y）+ Binance（BTC）
分析层: auto_briefing.py → 周期标签 + 置信度 + 信号等级
决策层: auto_briefing.py → 三层结论
  - TRANSITION/UNKNOWN/BEAR → 强制"观察"
  - BULL_MID A级 → 允许"分批建仓"
  - 第2层至少两条，第二条含当日具体数据
输出层: 四份报表 + START_HERE.html
治理层: governance_runtime.py + skill_gate.py
  拦截: 未批准执行 / 自验 / 错误线程
归因层: trade_record_log_v1.md + monthly_return_tracking_v1.md

## 三、运行原则

1. 信号质量优先于自动化程度
2. 置信度决定动作边界（A=可建仓/B=持有/C=只观察）
3. 每日流程: python scripts/auto_briefing.py → 输入Y → 人工决策
4. 四报表各自只回答一个问题，职责不重叠
5. 治理门控不可绕过

## 四、统一规范

数据缺失: 单项缺失→降级，不阻断 | 缺失≥3项→置信度降至C级
样本积累: 触发器≥10次 / 周期≥8次 才可统计
月度目标: 3.33%（40%÷12）
连续3个月低于目标 → 触发策略迭代

## 五、新模块继承规则

必须: 继承系统定位 / 前置检查 / 不新增下单接口 / 不存储密码 / 输出三元组

# 文件二：01_INVESTMENT_RULES.md（V6.1）
# AI投研明鉴规则库

版本: V6.1 | 目标: 40%是下限，趋势好时争取100%+

## 零、核心目标原则

年化40%是最低目标，不是最高目标。

收益目标分级:
  BULL_MID A级多维共振 → 争取100%+ → 积极进攻，仓位接近上限
  BULL_MID B级 → 争取60-80% → 稳健建仓
  TRANSITION B级 → 争取20-40% → 防守等待
  BEAR/UNKNOWN → 保本为先 → 保留现金

禁止以下伪稳健行为:
  - A级信号时仍只建半仓
  - 趋势明确时以"观察"拖延建仓
  - 以40%为目标自我设限

## 一、仓位管理

硬上限（任何周期不可突破）:
  美股IBKR: 50% | 日股SBI: 30% | A股富途: 30% | 加密BF: 10% | 现金: 最低10%

动态仓位:
  BULL_MID A级 → 70-80%（积极进攻）
  BULL_MID B级 → 50-70%（稳健建仓）
  TRANSITION B级 → 20-40%（轻仓观察）
  BEAR/C级 → 0-20%（防守保本）

基础规则:
  单一标的≤20% | 同方向≤50% | 禁止满仓（上限80%）| 回撤>10%→压缩至50%

信号等级对应:
  A级: 标准仓位，强趋势时可上调至硬上限-5%
  B级: 标准×0.7 | C级: 禁止新建

## 二、风险管理

止损: 单笔最大亏损账户净值3%，进场时确定，不事后移动

紧急模式触发（任意一条）:
  主要股指单日跌>5% / VIX单日跳升>30% / 用户手动宣布

紧急模式规则:
  周期降BEAR / 买入全停 / 现金强制30% / 连续5日无触发+用户确认才解除

年化目标边界:
  落后时可提高敞口，但不超过基准+10%
  绝对禁止: 杠杆超净值 / 突破硬上限 / TRANSITION/UNKNOWN/紧急期间提高敞口

## 三、买卖纪律

买入条件（全部满足）:
  1.周期BULL_MID或BULL_EARLY 2.置信度A（B可小仓试探）
  3.信号A或B 4.仓位未达上限 5.用户人工确认

禁止买入（任意一条）:
  置信度C / TRANSITION/UNKNOWN/BEAR / 5日涨>12% / 回撤>10% / 紧急中 / 数据失败≥3项

止盈:
  标准: +15%止盈50% / +30%止盈剩余50%
  强趋势: +15%仅止盈30%，留底仓追趋势
  周期失效: 不论盈亏强制全部止盈

## 四、周期分析（4维判断）

维度 / BULL / TRANSITION / BEAR:
  VIX: <18 / 18-23 / >23
  10Y美债: <4.0% / 4.0-4.7% / >4.7%
  SPX日涨跌: >+0.5% / -1%~+0.5% / <-1%
  BTC日涨跌: >+1% / -2%~+1% / <-2%

置信度: 3/4或4/4一致=A | 2/4=B | 1/4=C | 数据不足<3维降一级

强趋势窗口期（全部满足）:
  周期BULL_MID且置信度A / 4维100%共振 / SPX连续3日涨 / VIX<18且下行

周期失效:
  BULL_MID: VIX>25或SPX单日跌>3%
  TRANSITION: VIX>30或关键支撑跌破
  BEAR: VIX<20且SPX连续3日涨

## 五、归因原则（M1-M6）

M1: 周期定位错误 | M2: 资产配置错误 | M3: 买入触发错误
M4: 仓位计算错误 | M5: 止盈规则错误 | M6: 用户执行偏离

最低样本: 触发器≥10次 / 周期≥8次 / 单账户月度≥5笔 / 模块归因≥3次同类
不足时只输出: 样本不足（N次/最低M次），继续观察

未交易归因（必须记录）:
  A=触发未买 / B=应减未减 / C=违规买 / D=应止未止
  E=强趋势窗口期仓位过轻错失超额机会（V6.1新增）

# 文件三：02_REPORT_STANDARD.md
# 日报/周报/月报统一模板

版本: V6.0

## 一、日报（auto_briefing.py自动生成）

四份文件职责:
  latest_daily_report.md    → 今天发生了什么
  latest_direction_card.md  → 主线方向是否变化
  latest_risk_card.md       → 最大风险是什么
  latest_execution_card.md  → 今天该做什么

日报格式:
  # 今日日报 — YYYY-MM-DD
  昨日判断: 周期+置信度+结论 → 今日实际: SPX X% / VIX X → 验证结论
  今日市场变量: VIX / 10Y / SPX / BTC
  今日结论: [第0层] 周期:[标签] 置信度:[A/B/C]

方向卡格式:
  当前方向: 周期+置信度+结论
  失效条件: [具体条件]
  方向小结: 置信度X级——[是否可执行]

风险卡格式:
  最大风险: [第2层，至少两条]
  风险触发阈值: VIX警戒>25 / SPX警戒单日<-2%
  数据源状态: 四项各自OK或缺失

执行卡格式:
  唯一动作: [第1层]
  四账户边界: 富途/IBKR/SBI/BF各自动作
  执行前必须确认: 实时行情/持仓成本/可用现金/挂单
  下一步负责人/动作/线程

日报质量标准（违反任意一条=不通过）:
  1. 第0层和第1层不得矛盾
  2. TRANSITION/UNKNOWN/BEAR时第1层必须是"观察"
  3. 第2层至少两条，第二条含当日具体数据
  4. 日期必须为当日

## 二、周报（手动，每周五）

本周市场总结 / 判断复盘（准确率，样本≥8次才统计）/
交易摘要（周收益率，目标0.77%=40%÷52）/ 下周关注

## 三、月报（手动，每月末）

使用 monthly_return_tracking_v1.md
必填: 月初末资产/实际收益率/与3.33%差距/四账户贡献/M1-M6归因
禁止: 样本不足时输出有效/失效结论 / 归因只写"判断失误"
触发策略迭代: 连续3月<2% / 任一月亏损>-5% / 同模块连续3月主要拖累
