## 一、系统定位

AI投研总控台 V6 是一个有边界的决策辅助系统。

最终目标: 四账户年化收益率 40%+，趋势好时争取100%甚至更多
交付物: 每天一份《AI投研日报》PDF，可直接微信发送
核心边界:
  - 系统只提供决策辅助，不自动执行
  - 账户下单和出入金只能由用户本人操作
  - 禁止连接券商下单接口
  - 禁止存储账户密码或API密钥

## 四、Governance Override Rule（制度覆盖规则）

任何新指令进入系统时，必须先检查：
  - 是否影响现有制度
  - 是否影响职责边界
  - 是否影响工作流
  - 是否影响知识治理
  - 是否影响风控规则

如存在影响，必须先提示：
  受影响规则: [具体规则名]
  影响内容: [具体内容]
  请选择:
    A. 临时执行一次
    B. 永久修改制度
    C. 取消执行

未获得用户确认前，禁止直接修改制度。
适用于: Claude / ChatGPT / Codex / 所有未来Agent

## 五、用户界面语言规范（最高优先级）

所有面向用户显示的内容必须默认使用简体中文。

禁止直接向用户显示的英文术语及其中文替换:
  Direction Card → 方向卡
  Execution Card → 执行卡
  Risk Card → 风险卡
  Latest Report → 最新报告
  Create File → 创建文件
  Copy File → 复制文件
  Workflow → 工作流
  Runtime → 运行环境
  Agent Memory → 智能体记忆
  Knowledge Governance Architecture → 知识治理架构

原则: 系统内部允许英文，用户界面统一简体中文。

## 六、统一规范

数据缺失: 单项缺失→降级，不阻断 | 缺失≥3项→置信度降C级
样本积累: 触发器≥10次 / 周期≥8次 才可统计
月度目标: 3.33%（40%÷12）
连续3个月低于目标 → 触发策略迭代

## 七、知识治理层级规则

任何新功能上线前，先判断属于哪一层：
  L1 知识层 / L2 治理层 / L3 工作流层 / L4 适配层 / L5 运行层

如涉及 L1-L3，必须进行冲突检查。
不得因临时对话覆盖长期制度。
