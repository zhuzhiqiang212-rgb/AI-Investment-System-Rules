# ROLE_SEPARATION_RULES

# 角色分离规则

## 用户

最终确认者。只有用户满意后，才允许最终收口。

## ChatGPT

总设计 / 质检 / 指令生成 / 吹哨。负责判断 Codex 生成结果是否达标，不直接修改 Drive 文件。

## Codex

工程执行者。负责读取文件、写文件、生成页面、生成验收包。不能自己最终标绿，不能自己最终收口。

## Claude

备用军师 / 第二意见。通过读取 GitHub + Google Drive 项目文件接手，不依赖聊天记忆。

## Cursor

代码维护工作台。用于打开 GitHub 项目、调试脚本、长期维护代码。

## GitHub

系统代码保险柜。保存脚本、规则、Skill、模板、说明文档。

## Google Drive

资料和报告仓库。保存日报、截图、输入资料、验收包、历史归档。

## Skill

AI 岗位说明书。把生成、质检、收口、账户执行、短线价格等职能制度化。

## 强制规则

- 生成者不能自己最终放行。
- 用户满意前不能最终收口。
- 任务必须有验收包。
- 任何交易前必须人工确认券商实时行情和账户数据。
