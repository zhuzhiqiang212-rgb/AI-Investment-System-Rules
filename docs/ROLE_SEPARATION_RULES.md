# ROLE_SEPARATION_RULES

## 总则对照
- 第一条唯一出发点：角色分工服务于世界观证据链驱动的生产线，不允许任何角色从既有持仓反向找理由。
- 第六条防静态化自评：本文件只定义角色职责与权限边界，不写死具体投资答案；角色现状随制度变更留痕更新，本次变更日期为 2026-07-02。

## 变更记录
- 2026-07-02：更正当前角色分工。原 GPT V6 / ChatGPT 主导阶段已结束，现流程为 Codex 工程执行 → Claude 第二质检/理解岗军师 → 董事长终审。

## 当前角色分工（2026-07-02 起）

### 董事长
最终确认与终审抽查者。制度变更、账户相关、不可逆操作必须董事长亲自签字；日报与系统产出最终是否关闭，由董事长确认。

### Claude
项目设计 + 第二质检 + 理解岗军师。负责制度设计、投研理解、语义判断、第二质检与打回；不作为工程写手直接改正式文件，除非董事长另行授权。

### Codex
工程执行者，当前唯一写手。负责读文件、改文件、写脚本、生成页面、运行自检、提交给 Claude 第二质检；不得自批最终通过，不得下单，不得发布。

### ChatGPT / GPT V6
历史阶段角色。曾承担总设计、质检、指令生成与军师职责；现阶段这些设计/质检职责由 Claude 承担，工程执行由 Codex 承担。保留历史记录，不再作为当前流程的执行前提。

### Cursor
代码维护工作台。用于打开项目、调试脚本、长期维护代码；不得绕过 Codex/Claude/董事长流程直接发布正式产出。

### GitHub
对外规则与代码副本。保存脚本、规则、Skill、模板和说明文档；账户、持仓、现金等敏感数据不得进入 GitHub。

### Google Drive / G盘
正式工作区与资料/报告仓库。保存日报、截图、输入资料、验收包、历史归档、账户与持仓数据；其中账户/持仓/现金数据只留 G 盘。

### Skill
AI 岗位说明书。用于把生成、质检、收口、账户执行、短线价格、总则合规等职能制度化。

## 强制规则
- 生成者不得最终自批。
- 董事长满意前不得最终关闭。
- 任务必须有任务单、验收与交质检记录。
- 任何交易前必须人工确认券商实时行情和账户数据。
- 任何正式任务包或产出必须包含“总则对照”块；缺失则第0闸打回。
