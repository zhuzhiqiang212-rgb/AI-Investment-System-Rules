# RESEARCH_INTELLIGENCE_LANDING_V1：研究理解层落地方案 v1

> 状态：董事长 2026-07-02 已签字授权落地。  
> 边界：本文件只定义 v1 落地边界、每日流程、交接物与日报接入口。Codex 只做机械搬运＋预筛；“是否改今天动作”的理解判断由 Claude 理解岗完成。

## 1. v1 边界
- Codex 只做机械搬运＋预筛：读取已在知识库内的资料，去重，挑出今日新增或与当前持仓标的高相关的候选。
- Codex 不做全文自动读懂，不做全量资料理解，不做重要性判断。
- 第⑤栏“是否改今天动作”的判断由 Claude 理解岗逐条完成。
- v1 不追求把所有资料都读完，只保证候选入口干净、可追溯、不过量。
- 脚本只读只挑，不改资料、不下单、不发布。

## 2. 七个维度（每维 v1 浅做）
1. 主题分类。
2. 影响到谁。
3. 时间：今日变化 / 短期 / 长期 / 背景。
4. 方向：利好 / 利空 / 中性。
5. 是否改今天动作。
6. 置信度：高 / 中 / 低。
7. 去向：动作依据 / 仅背景 / Backlog。

## 3. 核心栏硬性写法
第⑤栏“是否改今天动作”必须写成：

```text
改/不改 ＋ 大白话“因为[发生了什么]→所以[对哪只/哪个动作什么影响]”
```

写不成“因为所以”人话的，判为不合格，打回。  
示例口径：
- 改：因为某公司订单证据今天变强→所以 NVDA 今日从“只观察”升级为“重点复核买入线”。
- 不改：因为这条只是长期行业背景→所以今天不改任何动作，只进 Backlog。

## 4. 每日四步流程
1. Codex 搬运＋预筛：去重、挑今日新增/高相关，候选不超过 15 条，不做重要性判断。
2. Claude 逐条七维打标，产出《今日研究理解包》。
3. 《今日研究理解包》接进日报 §研究理解。
4. 高价值源（老雷/深度研究）不进预筛淘汰，一律入候选，作为防漏兜底。

## 5. 《今日研究理解包》结构
顶部一句总结：

```text
今天有没有足以改动作的高置信结论：有/没有；一句话说明。
```

逐条结构：

```text
来源 | 一句话摘要 | 主题 | 影响谁 | 时间 | 方向 | 是否改今天动作(改/不改+因为所以) | 置信度 | 去向
```

## 6. v1 资料源
- v1 只接已在知识库的资料：老雷 / 湖水 / Drive。
- 新闻源等有稳定接口后再加入，标 v2。
- 读不到资料时，候选清单为空，并写明原因；不得伪造候选。

## 7. v1 交接物
- Codex 输出：`data/research/candidates_YYYYMMDD.json`。
- Claude 输出：《今日研究理解包》。
- 日报接入口：四账户日报 §研究理解。

## 8. 打回规则
- 候选清单无来源路径、无指纹、无筛选理由，打回。
- 候选超过 15 条且未说明高价值源兜底，打回。
- Claude 第⑤栏写不出“因为→所以”的人话，打回。
- Codex 脚本出现理解判断、买卖建议、下单动作，打回。
