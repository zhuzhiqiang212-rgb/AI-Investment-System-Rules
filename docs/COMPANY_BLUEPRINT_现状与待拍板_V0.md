## 0. 一句话大方向

把整个系统从"投资世界（一棵投资树）"**升级成一家"AI 投资决策公司"**：
日报 / 周报 / 月报 / 机会地图都只是**产品（产线末端流出来的东西）**；
真正的资产是那家**能持续造出高质量决策的公司**。
产品不好，不问"日报哪错了"，而问"哪个部门/工序失职了"。

## 1. 已确立的设计要点（对话中已谈定的调）

### 1.1 宪法第一原则：先有公司，后有人
- 公司定义的是**岗位说明书**（这个岗收什么、交什么、对什么负责、合格标准），不是"某个 AI"。
- Claude / Codex / GPT 只是今天**填进岗位的人**；换更强的 AI = 换人上岗，岗位说明书一字不改。
- 目的：**换 AI 不用重建公司**，因为公司按岗位设计、不按能人设计。

### 1.2 核心产品 = 决策本身，报表只是包装
- 公司真正生产的是"持续帮董事长实现长期资产增长的**投资决策**"。
- 同一个决策，可包装成日报给董事长看、包装成建议去执行、包装成月报去复盘。
- 好处：从此不再陷入"优化日报"——日报好不好取决于背后决策好不好，决策好不好取决于整条产线。

### 1.3 立体"经纬"组织，而非平面组织图
- **经线（纵向 = 业务流水线）**：
  世界观 → 国家战略 → 宏观 → 流动性 → 资金轮动 → 板块轮动 → 行业 → 公司 →
  竞争优势 → 估值 → 风险 → 机会 → 组合 → 今日动作 → 产品。
  每一层既**向下推导**、又**向上验证**，形成闭环。
- **纬线（横向 = 质量保障系统）**：法（标准）/ 测（验收）/ 环（协作反馈）/ PDCA，贯穿每一道工序。
- **交叉点 = 每道工序的质量关卡**：每层不只"加工"，还自带"按什么标准交、谁验收、不合格谁打回"。

### 1.4 质量靠"交接物 + 验收权 + 打回权"三件套
- 每道工序定死：交什么（有标准格式的产出物）、交得合不合格谁说了算（下游验收权）、
  不合格谁能打回重做（打回权）。
- **铁律**：下游有权拒收上游不合格产出物，且**不合格不得流入下一道工序**。
- 这是"质量部没拦住 → 产品不好"的根治办法：质量不是最后检一道，而是每道交接口都在检。

### 1.5 岗位"五件套"（含学习机制，这层决定公司能活多久）
每个岗位不止四件套，升级为五件套：
1. 岗位职责
2. SOP（标准作业）
3. 验收标准
4. 交接物
5. **学习机制（Learning Loop）** ← 新增，最关键

- **学习机制 ≠ PDCA**：PDCA 改"这次的错"（让**产品**不出错）；学习机制练"下次更强"（让**岗位**变强）。
  一句话：**PDCA 是改作业，学习机制是长本事。** 只改作业不长本事 → 永远在改同类的错
  （这正是过去几十轮日报反复返工的病根）。
- **公司同时生产两样东西**：对外的"决策"、对内的"能力"。能力沉淀在岗位里、不随 AI 走，
  换更强的 AI 直接继承老 AI 攒下的能力。
- **能力的落地形态 = 活的"岗位能力档案"**：经验库（历史判断+事后对错）+ 教训清单
  （从错误提炼的规则）+ 能力指标（命中率 / 重大风险漏报率 / 被打回率）。
- **闭环**：SOP 指导生产 → 结果进能力档案 → 档案反哺升级 SOP。SOP 是被学习机制持续改写的**活文件**。

### 1.6 落地路径：双轨并行，不推倒重来
1. **先定宪法**（现在）：定架构 / 岗位说明书 / SOP 标准 / 质量链 / 董事长保留权。**只定框架，不动现有任何东西。**
2. **再做资产盘点对照**：把现有日报 / Workflow / Skill / Evidence / 研究理解层，逐个映射到宪法里的岗位与工序，标"已有 / 需升级 / 缺失"。
3. **按优先级逐个升级**：不一次性重建；建议从"质量链 + SOP"这对开始（能立刻止住日报反复返工）。
- **现有资产处理原则**：不为升级而升级；先问"它属于哪个岗位？"属于 → 升级为正式岗位资产；
  不属于 → 允许淘汰。真正优秀的公司**东西越来越少、能力越来越强、组织越来越简单**。

### 1.7 已成型的子设计：Research Intelligence（研究理解层）
- 已出过《Research Intelligence 生产体系设计方案 V1》。它是公司"经线"上的一道工序。
- 定位：插在 Global Scan 之后、Research Evidence 之前，做**上游裁决者**。
- 补的唯一缺口：系统能扫、能登记、能生产，但没人判断"今天到底改变了哪些资产、够不够改今天动作"。
- 核心产物《今日研究理解包》九字段，灵魂字段是"**是否改变今天动作**"。
- 机制：预筛 + 指纹去重 + 长期观点台账（每天只看增量与变化）；双轨（收敛轨 + 高权重源兜底轨防漏）。
- ⚠️ 该方案原写三方为 Codex / Claude / GPT V6，需按新流程改（见第 3 节）。

## 2. 待董事长拍板的 5 个宪法级问题

动笔写正式 Company Blueprint 前，以下 5 题需董事长定调：

   —— 这是"宪法第一条"，最需要您拍。
4. **岗位五件套（含 Learning Loop）** —— 是否认可作为每个岗位的标准结构？
5. **落地路径（双轨并行：先宪法 → 资产盘点对照 → 逐个升级；第一步不动现有任何东西）** —— 这个节奏是否接受？
