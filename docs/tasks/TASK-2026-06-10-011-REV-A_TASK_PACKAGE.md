## 审批变更项（已合并，共5项）

1. 最终一句话结论
   简报第一眼必须看到今日一句话结论。
   禁止让用户先读五个区块。

2. 今日唯一动作
   每日最多一个动作（持有/减仓X%/观察/分批建仓），
   只能选一个，不能多个。

3. 最不能做
   固定存在一栏，列明当日禁止事项
   （禁止追高/禁止抄底/禁止加杠杆/禁止满仓等）。

4. 决策置信度
   输出 A/B/C 三级。
   C级时自动提示：仅观察，不执行。

5. 简报优先级高于日报
   用户先看简报，再决定是否打开日报/Direction/Risk/Execution。
   简报是入口，日报是详情。

## 交付内容

daily_decision_briefing_v1.md 必须包含：

1. 简报设计原则
   - 唯一入口原则：每天只有一份简报
   - 简报优先级高于日报（变更项5）：
     简报 → 决策前判断入口
     日报/Direction/Risk/Execution → 按需展开
   - 一句话结论必须第一眼可见（变更项1）
   - 今日唯一动作只能选一个（变更项2）
   - 最不能做栏位固定存在（变更项3）
   - 决策置信度 A/B/C，C级自动锁定（变更项4）
   - 6引擎串联顺序
   - 用户人工确认边界

2. 简报标准格式（顺序固定，不可调换）

   ┌ 第0行：今日一句话结论（变更项1）
   │ 格式：[结论] · 置信度[A/B/C]
   │ 示例：今天不新增高Beta · 置信度B
   │ C级时追加：⚠️ 仅观察，不执行

   ┌ 第1行：今日唯一动作（变更项2）
   │ 从以下四选一：
   │ 持有 / 减仓X% / 观察 / 分批建仓[账户+标的]
   │ 不得同时输出多个动作

   ┌ 第2行：今日最不能做（变更项3）
   │ 固定输出1-3条禁止事项
   │ 基于当日周期+信号状态自动生成
   │ 示例：禁止追高 / 禁止加杠杆 / 禁止满仓

   ┌ 区块A：今日市场定位
   │ 总周期标签 + 置信度
   │ 各资产子周期标签
   │ 配置有效期剩余天数

   ┌ 区块B：配置偏离检查
   │ 四账户当前实际比例 vs 目标比例
   │ 是否需要再平衡（YES/NO）
   │ EMERGENCY MODE 状态

   ┌ 区块C：今日信号汇总
   │ 四账户各自信号等级（A/B/C/无信号）
   │ 今日唯一动作来源（哪个账户哪个信号）

   ┌ 区块D：当前持仓风险检查
   │ 各持仓距止盈目标百分比
   │ 各持仓距止损失效条件百分比
   │ 加密仓位占总资产比例

   └ 区块E：归因预警
     近期策略失效预警（若有，含模块名称）
     本月胜率/盈亏比（含样本量）

3. 决策置信度判断规则（变更项4）
   A级（高置信）：
     - 周期置信度 A级
     - 信号等级 A级
     - 配置无偏离
     → 允许执行今日唯一动作
   B级（中置信）：
     - 周期置信度 B级，或信号等级 B级
     - 配置轻微偏离
     → 允许执行，但仓位降至 B级上限
   C级（低置信）：
     - 周期处于 TRANSITION / UNKNOWN
     - 或信号等级 C级
     - 或多个区块数据缺失
     → 自动输出：⚠️ 仅观察，不执行
     → 今日唯一动作强制替换为：观察

4. 今日一句话结论生成规则（变更项1）
   基于周期标签 + 信号等级 + 置信度
   自动选择以下模板之一：
   - BULL_MID + A级：允许分批布局[资产类别]
   - BULL_MID + B级：谨慎布局，控制仓位
   - BULL_LATE + 任意：减少新增，保护浮盈
   - BEAR + 任意：进入防守，持有现金
   - TRANSITION + 任意：今天进入观察期
   - UNKNOWN + 任意：证据不足，仅观察
   - EMERGENCY：今日进入紧急防守模式
   结论模板须覆盖七个周期状态 × 三个信号等级
   的完整矩阵（允许合并相似场景）

5. 今日唯一动作选择规则（变更项2）
   按以下优先级从高到低选一：
   优先级1：若有持仓触及止损失效条件 → 减仓/止损
   优先级2：若有 A 级买入信号且置信度≥B → 分批建仓
   优先级3：若配置严重偏离（>10%）→ 再平衡
   优先级4：若无以上情况 → 持有 或 观察
   置信度 C级时强制输出：观察

6. 最不能做生成规则（变更项3）
   固定检查以下条件，满足则输出对应禁止项：
   - 标的5日涨幅>12% → 禁止追高
   - 标的触及52周低点 → 禁止抄底
   - 账户回撤>10% → 禁止加仓
   - 当前周期 BULL_LATE/BEAR/EMERGENCY → 禁止加杠杆
   - 任何单一账户仓位>80% → 禁止满仓
   至少输出一条，最多三条。

7. 简报与日报/引擎的关系定义（变更项5）
   阅读顺序（强制）：
   第一步：打开 DAILY_DECISION_BRIEFING → 读第0/1/2行（30秒）
   第二步：若置信度 A/B → 按需展开区块A-E
   第三步：若需要更多细节 → 打开对应引擎文档
   第四步：最后才打开日报/Direction/Risk/Execution
   简报不替代日报，但简报先于日报。

8. 六引擎串联数据流
   步骤1：周期定位引擎 → 今日周期标签 + 置信度
   步骤2：资产配置引擎 → 配置偏离状态
   步骤3：买入触发器   → 各账户信号等级
   步骤4：仓位计算     → 若有信号，建议仓位比例
   步骤5：止盈系统     → 持仓止盈/止损状态
   步骤6：归因系统     → 失效预警（若有）
   步骤7（新）        → 整合输出第0/1/2行

9. 数据填充规范
   手动填入字段（用户每日操作）：
   VIX / 美债收益率 / 北向资金 / 四账户持仓市值
   自动获取字段（来自日报系统）：
   周期标签 / 配置比例 / 历史归因数据
   数据缺失降级规则：
   单字段缺失 → 对应行标注"数据待填"
   关键字段缺失（周期标签/信号等级）→ 置信度自动降为 C级

10. 真实示例一个
    完整模拟一天的简报输出：
    展示第0/1/2行生成过程 + 五区块完整内容
    展示 C级置信度时的自动锁定效果
