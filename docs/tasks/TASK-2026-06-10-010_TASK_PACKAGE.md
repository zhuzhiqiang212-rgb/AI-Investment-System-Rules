## 任务目标

为五模块决策体系设计策略归因系统，解决当前缺口：

赚了钱不知道是策略对了还是运气好。
亏了钱不知道是判断错了还是执行错了。
没有归因，系统无法自我进化。

与已完成五模块的联动：
  买入触发器  → 进场决策记录
  仓位计算    → 进场规模记录
  止盈系统    → 出场决策记录
  周期定位    → 周期背景记录
  资产配置    → 配置背景记录
      ↓
  策略归因系统 → 对每笔交易和每个策略模块
                进行结果拆解和进化建议

## 交付内容

strategy_attribution_system_v1.md 必须包含：

1. 归因系统设计原则
   - 归因目标：识别哪些模块在赚钱，哪些在亏钱
   - 归因层级：交易级 / 策略级 / 系统级
   - 归因与复盘的关系（归因是复盘的定量基础）
   - 归因数据来源（五个已完成模块的输出记录）
   - 用户人工确认边界（归因只输出结论，不自动调整策略）

2. 交易记录标准格式
   每笔交易必须记录以下字段（用于归因输入）：
   - 交易 ID / 账户 / 标的 / 方向
   - 进场日期 / 进场价格 / 进场仓位比例
   - 触发信号等级（A/B/C）
   - 触发时的周期标签 + 置信度
   - 触发时的资产配置背景
   - 出场日期 / 出场价格 / 出场原因
     （止盈第一目标 / 止盈第二目标 / 止损 /
      强制止盈 / 信号降级减仓 / 周期失效）
   - 实现盈亏（绝对值 + 占账户比例）
   - 持仓天数

3. 三层归因框架

   层级1：交易级归因
   对每笔交易拆解：
   - 进场判断是否正确（触发条件事后验证）
   - 仓位大小是否合适（对结果的放大/缩小效果）
   - 出场时机是否合适（早出/晚出/按规则出）
   - 周期背景是否匹配（顺周期/逆周期）
   - 归因结论：策略正确 / 执行偏差 / 运气成分

   层级2：策略级归因
   对每个模块的历史表现统计：
   - 买入触发器：A/B/C各等级胜率和盈亏比
   - 仓位计算：不同仓位大小对总收益的贡献
   - 止盈系统：第一/第二止盈目标命中率
   - 周期定位：各周期标签下的平均收益率
   - 资产配置：各账户对总收益的贡献比例

   层级3：系统级归因
   整体分析：
   - 总胜率 / 总盈亏比
   - 年化收益率 vs 40% 目标的差距来源
   - 哪个模块贡献最大 / 哪个模块拖累最大
   - 策略失效预警：某模块连续3次判断失误时触发

4. 策略失效预警规则
   当以下任意情况出现时，自动输出预警：
   - 同一触发器信号等级连续3次亏损
   - 同一周期标签下连续2次判断失误
   - 任一账户单月收益率连续2个月
     落后对应目标超过 2%
   预警内容：
   - 失效模块名称
   - 失效证据（具体交易记录）
   - 建议动作（暂停该模块 / 降级置信度 / 人工复核）

5. PDCA 归因闭环格式
   每周/每月必须执行一次归因复盘：
   Plan：本期策略预期
   Do：实际执行记录
   Check：归因拆解结论
   Act：下期策略调整建议
   输出格式须与日报 V2 PDCA 模块衔接。

6. 真实示例两个
   示例1：一笔完整的 NVDA 交易归因
   （从进场记录到出场归因，三层拆解）
   示例2：月度策略级归因报告
   （买入触发器和周期定位的历史表现统计）

7. 日报 V2 归因字段格式
   须在日报第三屏 PDCA 区域显示：
   - 本月已归因交易数量
   - 当前胜率 / 盈亏比
   - 最近一次策略失效预警（若有）
   - 距年化40%目标的差距归因（一句话）
