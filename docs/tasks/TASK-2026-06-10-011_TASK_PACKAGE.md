## 任务目标

把六个已完成引擎串联成用户每天打开的
唯一一份"当日决策前判断简报"。

解决当前核心缺口：
六个引擎是六份孤立文档，
用户每天仍需手动查阅多个文件。
本任务定义：
每日简报的标准输出格式、
六引擎的查表顺序与数据流、
用户3分钟完成决策前判断的完整流程。

## 交付内容

daily_decision_briefing_v1.md 必须包含：

1. 简报设计原则
   - 唯一入口原则：每天只有一份简报
   - 六引擎串联顺序（数据流方向）
   - 3分钟决策前判断目标
   - 用户人工确认边界

2. 六引擎串联数据流
   定义从周期定位到最终动作的完整查表顺序：
   步骤1：周期定位引擎 → 今日周期标签 + 置信度
   步骤2：资产配置引擎 → 当前配置是否偏离目标
   步骤3：买入触发器   → 各账户今日信号等级
   步骤4：仓位计算     → 若有信号，建议仓位比例
   步骤5：止盈系统     → 当前持仓止盈状态检查
   步骤6：归因系统     → 近期策略失效预警（若有）

3. 当日决策前判断简报标准格式
   五个必填区块：
   区块1：今日市场定位
     - 总周期标签 + 置信度
     - 各资产子周期标签
     - 配置有效期剩余天数
   区块2：配置偏离检查
     - 四账户当前实际比例 vs 目标比例
     - 是否需要再平衡（YES/NO）
     - EMERGENCY MODE 状态
   区块3：今日信号汇总
     - 四账户各自信号等级（A/B/C/无信号）
     - 多信号优先级排序规则
     - 今日唯一动作（最高优先级一项）
   区块4：当前持仓风险检查
     - 各持仓止盈状态（距目标价百分比）
     - 各持仓止损状态（距失效条件百分比）
     - 加密仓位占总资产比例（实时检查上限）
   区块5：归因预警
     - 近期策略失效预警（若有，含模块名称）
     - 本月胜率/盈亏比（含样本量）

4. 数据填充规范
   定义每个区块需要的实时数据字段：
   - 哪些数据需要用户每日手动填入
   - 哪些数据可从已有报告系统自动获取
   - 数据缺失时的降级处理规则
     （数据缺失 → 对应区块标注"数据待填"，
      不影响其他区块输出）

5. 多信号优先级排序规则
   同日出现多个买入信号时：
   - A级信号优先于B/C级
   - 同等级时：美股 > 加密 > 日股 > A股
     （基于流动性和执行便利性）
   - 同账户多标的：按仓位空间排序
   - 每日最多执行2个信号（防止过度交易）

6. 日报 V2 与简报的关系定义
   简报（本任务）= 决策前判断（早盘前完成）
   日报 V2     = 决策后记录（含当日执行动作）
   两者的区别、各自的触发时机、数据共享规则

7. 真实示例一个
   完整模拟一天的简报输出：
   从步骤1到步骤6的完整查表过程，
   输出五个区块的完整内容，
   展示用户如何在3分钟内完成决策前判断
