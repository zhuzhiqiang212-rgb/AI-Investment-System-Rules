## 审批变更项（已合并，共4项）

1. 未交易归因
   不仅归因已执行交易，还须记录并归因：
   - 触发信号但未买入
   - 应减仓但未减仓
   - 不应买入但买入了
   - 应止盈但未止盈
   识别"错过机会"和"违规执行"两类偏差。

2. 人工干预归因
   区分系统建议与用户实际操作，
   记录偏离原因，归因到：
   系统错误 / 用户偏离 / 数据不足 / 执行延迟。

3. 模块责任归因
   亏损或错过必须归因到具体模块：
   周期定位错误 / 资产配置错误 / 买入触发错误 /
   仓位计算错误 / 止盈规则错误 / 用户执行偏离。
   禁止只写"判断失误"。

4. 复盘最低样本规则
   样本不足时禁止输出策略有效/失效结论，
   只能输出"样本不足，继续观察"。
   须定义各类归因的最低样本量阈值。

## 交付内容

strategy_attribution_system_v1.md 必须包含：

1. 归因系统设计原则
   - 归因目标与层级（交易级/策略级/系统级）
   - 未交易归因的必要性（变更项1）
   - 人工干预归因框架（变更项2）
   - 模块责任归因体系（变更项3）
   - 最低样本规则（变更项4）
   - 归因数据来源（五模块输出记录）
   - 用户人工确认边界

2. 交易记录标准格式（已执行交易）
   每笔交易必须记录：
   - 交易ID / 账户 / 标的 / 方向
   - 进场日期 / 进场价格 / 进场仓位比例
   - 触发信号等级（A/B/C）
   - 触发时周期标签 + 置信度
   - 触发时资产配置背景
   - 系统建议操作 / 用户实际操作（变更项2）
   - 偏离原因（若存在偏离）（变更项2）
   - 出场日期 / 出场价格 / 出场原因
   - 系统建议出场时机 / 用户实际出场时机（变更项2）
   - 实现盈亏（绝对值 + 占账户比例）
   - 持仓天数
   - 归因结论（变更项3：模块责任归因）

3. 未交易记录标准格式（变更项1）
   以下四类未交易事件须强制记录：
   类型A：触发信号但未买入
     - 触发日期 / 标的 / 信号等级
     - 未买入原因（用户主动放弃/漏看/其他）
     - 事后价格走势（N日后涨跌幅）
     - 归因结论：正确放弃 / 错过机会

   类型B：应减仓但未减仓
     - 触发日期 / 标的 / 减仓信号来源
     - 未减仓原因
     - 事后结果（持仓盈亏变化）
     - 归因结论：正确持有 / 违规未执行

   类型C：不应买入但买入了
     - 买入日期 / 标的 / 违反的不买入条件
     - 用户买入原因
     - 实现盈亏
     - 归因结论：违规执行（注明违反哪条规则）

   类型D：应止盈但未止盈
     - 触发日期 / 标的 / 止盈信号来源
     - 未止盈原因
     - 事后结果（最终出场价 vs 应止盈价）
     - 归因结论：正确持有 / 错失止盈

4. 人工干预归因框架（变更项2）
   每笔交易和未交易事件须标注：
   系统建议：[系统输出的建议操作]
   用户操作：[用户实际执行的操作]
   一致性：一致 / 偏离
   若偏离，归因到以下四类之一：
   - 系统错误（系统建议有误，用户判断正确）
   - 用户偏离（用户未遵守系统建议）
   - 数据不足（归因时缺乏关键数据）
   - 执行延迟（建议正确但执行时间偏差）
   人工干预归因汇总：
   每月统计用户偏离次数及偏离后的平均收益影响。

5. 模块责任归因体系（变更项3）
   每笔亏损或错过机会须归因到具体模块：
   - 周期定位错误：周期标签事后证明错误
   - 资产配置错误：配置比例不适合当时周期
   - 买入触发错误：触发条件设计存在缺陷
   - 仓位计算错误：仓位大小与风险不匹配
   - 止盈规则错误：止盈目标设计存在缺陷
   - 用户执行偏离：系统建议正确，用户未执行
   禁止：归因结论只写"判断失误"而不指向具体模块。
   允许：一笔交易归因到多个模块（注明各模块责任权重）。

6. 复盘最低样本规则（变更项4）
   各类归因的最低样本量阈值：
   - 单一触发器信号等级（A/B/C）胜率：最少10次
   - 单一周期标签下收益统计：最少8次
   - 单一账户月度归因：最少5笔交易
   - 模块责任归因：最少3次同类错误
   - 人工干预偏离统计：最少5次偏离
   样本不足时输出：
     "样本不足（当前N次，最低需M次），继续观察。"
   禁止在样本不足时输出：
     "该策略有效" / "该策略失效" / 任何确定性结论。

7. 三层归因框架
   层级1：交易级归因
   层级2：策略级归因（五模块各自历史表现统计）
   层级3：系统级归因（总胜率/盈亏比/年化差距来源）

8. 策略失效预警规则
   触发条件（任意一条）：
   - 同一触发器信号等级连续3次亏损
   - 同一周期标签下连续2次判断失误
   - 任一账户单月收益率连续2个月落后目标超2%
   预警须指向具体模块（变更项3），
   样本不足时预警降级为"观察提示"而非"失效警告"。

9. PDCA 归因闭环格式
   周/月必须执行一次：
   Plan / Do / Check（含未交易归因 + 人工干预统计）/
   Act（下期策略调整建议，须指向具体模块）

10. 真实示例两个
    示例1：一笔 NVDA 完整归因
    含：已执行交易三层拆解 + 模块责任归因
    示例2：月度归因报告
    含：策略级统计 + 未交易事件汇总 +
        人工干预偏离统计 + 样本量检查

11. 日报 V2 归因字段格式
    第三屏 PDCA 区域显示：
    - 本月已归因交易数量 / 未交易事件数量
    - 当前胜率 / 盈亏比（标注样本量）
    - 人工干预偏离次数及影响
    - 最近策略失效预警（若有，含模块名称）
    - 距年化40%目标差距归因（一句话，含模块责任）
