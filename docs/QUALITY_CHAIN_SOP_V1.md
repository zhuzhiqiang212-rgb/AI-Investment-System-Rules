# QUALITY_CHAIN_SOP_V1：数据→估值→风险→动作→日报 四道闸门

> 状态：V1 落档。董事长已授权先从质量链＋SOP 起。  
> 来源口径：本 V1 把已落档的日报标准 V7、Q0/Q1 今日动作质检线、TASK-007 成本口径、TASK-010 富途现金口径折成自动闸门。TASK-013 已按 Claude 第二质检线确认的 v1 口径固化四个尾巴：不加行情时间差阈值、不强制补真估值、风险阈值不量化、图文美化不纳入硬打回。

## 总原则
- 每道闸门必须有：交接物、验收标准、打回条件。
- 每道闸门先由脚本只读检查；脚本不改数据、不改日报、不下单、不发布。
- 任一闸门打回，日报不得进入 dashboard。
- 占位、缺数据可以存在，但必须诚实标注，并按“不可下单/待补/人工核对”规则处理。

## 闸门① 数据→估值

### 交接物
- `data/accounts/unified_holdings_latest.json`
- 当日四账户日报 HTML
- 现金、成本、数据等级、来源时间戳与待补项

### 验收标准（可逐条打勾）
- [ ] 统一持仓库 JSON 可解析，且 `entries` 非空。
- [ ] 统一持仓库 `summary` 含 `holdings_total_usd`、`known_cash_usd`、`known_assets_total_usd`。
- [ ] 每条持仓有 `account`、`ticker`、`quantity`、`price`、`currency`、`market_value_usd`。
- [x] 持仓数量与 USD 市值不得为空；逐币/逐股价格缺失时，只要有正市值、来源标注与不可下单口径，v1 可作为“标的级市值”放行。
- [ ] 数据来源/等级可追溯：A 级 OpenD、B+ OCR、待补项必须能区分。
- [ ] 富途现金若已读到，不得继续标“富途现金待补”；HKD/CNH 缺 FX 只能列原币并标待补 FX。
- [x] v1 不加行情时间差阈值：数据→估值只校验来源、数量、市值、待补与不可下单口径。

### 打回条件
- JSON 不能解析或核心字段缺失。
- 持仓数量/USD 市值缺失却未标待补；或逐币/逐股价格缺失但没有正市值和来源标注。
- 用 0、现价、旧截图冒充缺失数据。
- 富途现金已接入后仍把账户现金整体标 pending。

## 闸门② 估值→风险

### 交接物
- 日报估值/机会区块。
- 已知真估值标的清单。
- 占位估值、待补估值、不可下单兜底说明。
- 统一持仓库成本与盈亏覆盖状态。

### 验收标准（可逐条打勾）
- [ ] 日报必须区分“真估值”和“占位/待补估值”。
- [ ] 真估值标的必须有明确可信度或情景说明。
- [ ] 占位估值不得作为买卖依据，必须带“不可下单/不可自动下单/待补真估值”兜底。
- [ ] 持仓成本缺口必须标待补，不得用 0 或现价冒充。
- [ ] 占位估值存在但兜底完整时，闸门状态为“PASS_WITH_GUARDRAIL（靠不可下单兜底放行）”，不是完全通过。
- [x] v1 不强制任何标的补真估值；占位估值必须不可下单兜底并进入待补。

### 打回条件
- 占位估值未标“不可下单/待补”。
- 把占位价格写成正式买卖计划。
- 成本缺失被填 0 或用现价冒充。
- 风险分析引用了未兜底的占位估值。

---

## 闸门③ 风险→今日动作

### 交接物
- 日报风险区块。
- 今日大局/今日动作/禁止动作。
- 风险升级、缓和、人工核对项。

### 验收标准（可逐条打勾）
- [ ] 今日是否交易必须明确，不能只写泛泛观察。
- [ ] 风险区块必须说明主要风险来源。
- [ ] 必须有禁止动作或“不做什么”。
- [ ] 高风险/高 Beta/集中度敞口必须和今日动作相连。
- [ ] 数据等级不足时，必须说明下单前人工核对。
- [x] 风险阈值 v1 不量化；先检查风险是否连到今日动作、禁止动作与人工核对。

### 打回条件
- 风险存在但没有动作或禁止动作。
- 动作是通用原则，不是今天动作。
- 数据等级不足却给出可直接下单结论。

## 闸门④ 今日动作→日报产品

### 交接物
- 当日最终日报 HTML。
- 统一持仓库与质检输出。
- 日报标准 V7 红线。

### 验收标准（可逐条打勾）
- [ ] 日报第一屏能看到今天该不该动。
- [ ] 重要持仓有“但反过来想/翻盘信号”或等价反方说明。
- [ ] 报告包含风险、机会、账户与现金、缺口与禁止。
- [ ] 非实时/截图/OCR/占位数据必须显式标注。
- [ ] 报告必须声明非下单指令、最终决策在董事长。
- [x] 图文美化不纳入 v1 硬打回；本闸门只检查动作可见、反方、来源标注、禁止与声明。

### 打回条件
- 日报看不出今天该不该动。
- 重要持仓无反方/翻盘信号。
- 缺数据、占位、OCR 未显式标注。
- 报告给出自动下单或可直接下单口径。
