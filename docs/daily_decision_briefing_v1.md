## 第一部分：设计原则

### 1.1 简报定位

简报（本规范）= 决策前判断（早盘前完成）
日报 V2      = 决策后记录（含当日执行动作）

### 1.2 优先级

本文件 > 日报 > Direction Card > Risk Card > Execution Card
用户先看简报，再决定是否打开其他文件。

### 1.3 唯一入口

UNIQUE_ENTRY_POINT: daily_briefing_template_v1.md
禁止从日报/Direction/Risk 进入，必须从模板开始。

### 1.4 目标

用户3分钟内完成决策前判断。
auto_briefing.py 运行后约30秒完成。

---

## 第二部分：简报标准格式

### 固定输出顺序（不可调换）

```
第0行：今日一句话结论 + 置信度
       C级时追加：⚠️ 仅观察，不执行

第1行：今日唯一动作（四选一）
       持有 / 减仓X% / 观察 / 分批建仓[账户+标的]

第2行：今日最不能做（1-3条）
       基于当日周期+信号状态自动生成

区块A：今日市场定位
       总周期标签 + 置信度 + 各资产子周期

区块B：配置偏离检查
       四账户实际比例 vs 目标比例 + EMERGENCY状态

区块C：今日信号汇总
       四账户各自信号等级（A/B/C/无信号）

区块D：当前持仓风险检查
       距止盈% / 距止损% / 加密占比

区块E：归因预警
       本月胜率/盈亏比（含样本量）/ 失效预警
```

---

## 第三部分：决策置信度

| 等级 | 条件 | 操作权限 |
|------|------|---------|
| A级 | 周期置信度A + 信号A + 无配置偏离 | 允许执行唯一动作 |
| B级 | 周期置信度B或信号B + 轻微偏离 | 允许执行，仓位降至B级上限 |
| C级 | TRANSITION/UNKNOWN或信号C或多项数据缺失 | 自动锁定为"观察"，不执行 |

---

## 第四部分：今日一句话结论模板

| 周期+置信度 | 结论模板 |
|-----------|---------|
| BULL_MID A | 允许分批布局AI核心 |
| BULL_MID B | 谨慎布局，控制仓位 |
| BULL_LATE 任意 | 减少新增，保护浮盈 |
| BEAR 任意 | 进入防守，持有现金 |
| TRANSITION 任意 | 今天进入观察期 |
| UNKNOWN 任意 | 证据不足，仅观察 |
| EMERGENCY 任意 | 今日进入紧急防守模式 |

---

## 第五部分：今日唯一动作选择规则

优先级（从高到低，选一）：
1. 持仓触及止损失效条件 → 减仓/止损
2. A级买入信号且置信度≥B → 分批建仓
3. 配置严重偏离（>10%） → 再平衡
4. 无以上情况 → 持有 或 观察
5. 置信度C级 → 强制输出"观察"

---

## 第六部分：今日最不能做生成规则

自动检查以下条件：
- 标的5日涨幅>12% → 禁止追高
- 标的触及52周低点 → 禁止抄底
- 账户回撤>10% → 禁止加仓
- 周期BULL_LATE/BEAR/EMERGENCY → 禁止加杠杆
- 任何单一账户仓位>80% → 禁止满仓

至少输出一条，最多三条。

---

## 第七部分：NEXT 三元组（强制输出）

每次简报末尾必须输出：

```
NEXT_OWNER  : 用户
NEXT_ACTION : [基于置信度和唯一动作的具体建议]
NEXT_THREAD : AI投研总控台 + 正式日报生产
```

---

## 第八部分：数据缺失降级规则

| 缺失字段 | 降级处理 |
|---------|---------|
| VIX | 步骤1标注"数据待填"，不阻断 |
| 美债 | 同上 |
| Positioning | 周期置信度自动降B级 |
| Onchain | 加密信号降C级 |
| 必填字段缺失≥3项 | 整体置信度降C级 |

---

## 第九部分：简报与六引擎的关系

```
周期定位引擎  → 步骤1：今日周期标签+置信度
资产配置引擎  → 步骤2：配置偏离状态
买入触发器    → 步骤3：各账户信号等级
仓位计算引擎  → 步骤4：建议仓位比例
止盈系统      → 步骤5：持仓止盈止损状态
归因系统      → 步骤6：失效预警
    ↓
auto_briefing.py 整合输出第0/1/2行
```
