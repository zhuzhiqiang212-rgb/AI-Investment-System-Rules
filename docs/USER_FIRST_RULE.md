# USER-FIRST RULE（用户优先规则）
# V6 最高优先级治理规则
# 生效时间: 2026-06-15 JST
# 制定依据: 用户明确指令

---

## 核心原则

用户不是开发人员。
用户只负责三件事：
  1. 打开日报
  2. 查看内容
  3. 做投资决策

---

## 禁止要求用户执行的操作

以下操作属于 Claude / Codex 职责，禁止转交给用户：

  ❌ 打开 PowerShell / 命令行
  ❌ 输入 Python 命令
  ❌ 运行任何脚本（包括 auto_briefing.py）
  ❌ 输入 Y / N 确认程序运行
  ❌ 手工验证程序输出
  ❌ 阅读日志文件
  ❌ 阅读代码
  ❌ 阅读执行卡
  ❌ 手动复制/移动文件
  ❌ 手动下载/上传文件

---

## 正式日报生产流程（用户零操作版）

每天早盘前，系统自动完成：

  Claude:
    检查 Drive 资讯更新
    检查 GitHub 规则更新
    检查知识库一致性
    ↓
  Codex:
    自动运行 auto_briefing.py
    自动生成 00_今日日报.pdf
    自动写入桌面\股票分析与研究\
    ↓
  用户:
    打开 00_今日日报.pdf
    阅读内容
    做投资决策
    结束

---

## 自检规则（Claude 必须执行）

每次准备要求用户操作时，先问自己：

  这是用户动作还是系统动作？

  如果是系统动作：
    → 设计自动化，不要求用户操作
    → 发出给 Codex 的执行卡

  如果确实是用户动作（如：开盘后查看持仓）：
    → 说明为什么不能自动化
    → 提出未来自动化改造方案

## 违规判定

Claude 在以下情况违反 USER-FIRST RULE：

  输出"请运行 python xxx.py"
  输出"请打开 PowerShell"
  输出"请输入 Y 确认"
  输出"请手动复制文件"
  输出"请截图发给我验证"（技术验证类）

违规时，用户可直接回复：
  "这是用户动作还是系统动作？"
  Claude 必须立即停止，重新设计自动化方案。
