# Position Sizing Engine V1

来源任务：TASK-2026-06-10-006 REV-A
部署任务：TASK-2026-06-10-017
用途：Daily Briefing V2 查表区步骤4，将A/B/C信号转换为建议仓位比例。

## 1. 仓位计算设计原则

本引擎只输出建议仓位比例，不生成交易指令，不自动下单。
执行前必须人工确认券商实时行情、现金、持仓、成本、挂单。

规则：
- 与A/B/C信号等级联动。
- 账户回撤越大，仓位越自动压缩。
- 总仓位上限优先于单标的信号。
- 绝对禁止满仓规则：任何账户、任何A级信号，单笔单标的不得超过账户净值固定上限。
- 账户现金保留规则：每个账户必须保留最低现金比例。

## 2. 四账户各自仓位计算模板

### 富途

- 基础仓位公式：A级=基准仓位X%，B级=基准仓位X%的一半，C级=0。
- 最低现金保留比例：X%。
- 单标的满仓上限：不得超过账户净值X%。
- 回撤压缩系数：账户回撤越大，建议仓位按比例压缩。
- 波动率调整：高beta标的自动下调。
- 单标的集中度上限：不得超过账户单标的上限。
- 加仓规则：仅在信号不降级、现金确认、无重复挂单时允许。
- 信号降级减仓规则：A→B减至B级上限；B→C停止新增并评估降低；跌破失效条件进入风险处理。

### IBKR

- 基础仓位公式：A级/B级/C级分别映射到建议比例区间。
- 最低现金保留比例：X%，保证金账户额外保留安全垫。
- 单标的满仓上限：不得超过账户净值X%。
- 回撤压缩系数：回撤触发后按风险等级压缩。
- 波动率调整：COIN/MSTR/IREN等高beta降档。
- 单标的集中度上限：核心AI与高beta分别设上限。
- 加仓规则：需要现金、保证金、挂单人工确认。
- 信号降级减仓规则：A→B降风险；B→C仅保留观察；跌破失效条件必须人工复核。

### SBI

- 基础仓位公式：A级/B级/C级对应日股建议比例区间。
- 最低现金保留比例：X%买付余力。
- 单标的满仓上限：不得超过账户净值X%。
- 回撤压缩系数：账户回撤或日元波动放大时压缩。
- 波动率调整：9984/6857/8035连续拉升时降档。
- 单标的集中度上限：日股AI链和金融保险链分开控制。
- 加仓规则：未约定订单必须确认。
- 信号降级减仓规则：A→B减至B级上限；B→C暂停新增；跌破失效条件进入防守。

### Bitflyer

- 基础仓位公式：A级/B级/C级对应BTC/ETH建议比例区间。
- 最低现金保留比例：X%日元余额。
- 单标的满仓上限：不得超过账户净值X%。
- 回撤压缩系数：加密回撤和波动同步压缩。
- 波动率调整：BTC/ETH资金费率过热时降档。
- 单标的集中度上限：BTC核心和ETH波段分开限制。
- 加仓规则：余额和挂单人工确认后才可计算。
- 信号降级减仓规则：A→B降低波动；B→C不新增；跌破失效条件转防守。

## 3. 回撤压缩系数

回撤压缩系数用于防止亏损扩大时继续机械加仓。
格式：
建议仓位 = 基础仓位 × 信号系数 × 回撤压缩系数 × 波动率系数。

若账户进入明显回撤，压缩系数下降。
若信号降级，优先执行信号降级规则。

## 4. 与买入触发器的衔接格式

触发器输出信号等级 → 仓位引擎输出建议仓位百分比 → 用户人工确认后执行。

## 5. 真实示例

### 美股
NVDA信号A级但账户回撤扩大：基础仓位需乘以回撤压缩系数，不允许满仓。

### 日股
9984趋势强但热度高：信号从A降B，仓位降至B级上限。

### A股
半导体设备B级观察：只输出小比例建议，现金不足则不执行。

### 加密
BTC信号B级但资金费率过热：波动率系数下调，建议仓位下降。
