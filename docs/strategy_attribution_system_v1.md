## 第一部分：设计原则

### 1.1 归因目标

赚了钱不知道是策略对了还是运气好。
亏了钱不知道是判断错了还是执行错了。
没有归因，系统无法自我进化。

### 1.2 归因层级

- 层级1：交易级归因（每笔交易）
- 层级2：策略级归因（每个模块历史表现）
- 层级3：系统级归因（总胜率/盈亏比/年化差距）

### 1.3 未交易归因（必须记录）

不仅归因已执行交易，还须记录：
- 触发信号但未买入
- 应减仓但未减仓
- 不应买入但买入了
- 应止盈但未止盈

### 1.4 人工干预归因

区分系统建议与用户实际操作，归因到四类：
- 系统错误
- 用户偏离
- 数据不足
- 执行延迟

### 1.5 模块责任归因

亏损或错过必须归因到具体模块（禁止只写"判断失误"）：
- 周期定位错误
- 资产配置错误
- 买入触发错误
- 仓位计算错误
- 止盈规则错误
- 用户执行偏离

### 1.6 最低样本规则

样本不足时禁止输出策略有效/失效结论，只能输出：
"样本不足（当前N次，最低需M次），继续观察。"

最低样本量阈值：
- 触发器各等级胜率：最少10次
- 周期标签下收益统计：最少8次
- 单账户月度归因：最少5笔
- 模块责任归因：最少3次同类错误

## 第二部分：交易记录标准格式

| 2. 账户 | 进场时 |
| 3. 标的 | 进场时 |
| 4. 方向 | 进场时 |
| 5. 进场日期 | 进场时 |
| 6. 进场价格 | 进场时 |
| 7. 进场仓位比例 | 进场时 |
| 8. 触发信号等级（A/B/C） | 进场时 |
| 9. 触发时周期标签+置信度 | 进场时 |
| 10. 系统建议/偏离 | 进场时 |
| 11. 出场日期 | 出场时 |
| 12. 出场价格 | 出场时 |
| 13. 出场原因 | 出场时 |
| 14. 实现盈亏 | 出场时 |

类型A：触发未买入
  日期 / 标的 / 等级 / 未买原因 / 事后涨跌 / 归因

类型B：应减未减
  日期 / 标的 / 信号来源 / 未减原因 / 事后变化 / 归因

类型C：违规买入
  日期 / 标的 / 违反规则 / 买入原因 / 盈亏 / 归因

类型D：应止未止
  日期 / 标的 / 止盈信号 / 未止原因 / 事后结果 / 归因

## 第三部分：三层归因框架

### 层级1：交易级归因

每笔交易拆解：
- 进场判断是否正确
- 仓位大小是否合适
- 出场时机是否合适
- 周期背景是否匹配
- 归因结论：策略正确 / 执行偏差 / 运气成分

### 层级2：策略级归因

| 模块 | 统计指标 |
|------|---------|
| 买入触发器 | A/B/C各等级胜率和盈亏比 |
| 仓位计算 | 不同仓位大小对总收益的贡献 |
| 止盈系统 | 第一/第二止盈目标命中率 |
| 周期定位 | 各周期标签下的平均收益率 |
| 资产配置 | 各账户对总收益的贡献比例 |

### 层级3：系统级归因

- 总胜率 / 总盈亏比
- 年化收益率 vs 40% 目标的差距来源
- 哪个模块贡献最大 / 哪个模块拖累最大
- 策略失效预警（连续3次失误触发）

---

## 第四部分：策略失效预警

触发条件（任意一条）：
- 同一触发器信号等级连续3次亏损
- 同一周期标签下连续2次判断失误
- 任一账户单月收益连续2个月落后目标超2%

预警须指向具体模块，样本不足时降级为"观察提示"。

---

## 第五部分：PDCA 归因闭环

每周/每月执行一次：
- Plan：本期策略预期
- Do：实际执行记录
- Check：归因拆解结论（含未交易归因+人工干预统计）
- Act：下期策略调整建议（须指向具体模块）

---

## 第六部分：日报 V2 归因字段

第三屏 PDCA 区域显示：
- 本月已归因交易数量 / 未交易事件数量
- 当前胜率 / 盈亏比（标注样本量）
- 人工干预偏离次数及影响
- 最近策略失效预警（若有，含模块名称）
- 距年化40%目标差距归因（一句话，含模块责任）
