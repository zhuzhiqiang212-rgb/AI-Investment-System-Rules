# AI投研体系 工作目标清单 V1.0
# MASTER_GOAL_LIST

生效时间: 2026-06-11 JST
制定依据: AI投研体系总纲 V2
最终目标: 四账户年化收益率 40%+
账户操作原则: 只有用户本人可以下单和出入金，系统只提供决策辅助

---

## 核心原则

1. 系统交付：有边界的决策辅助，不是自动执行
2. 账户安全：禁止连接下单接口，禁止自动下单，禁止读取账户密码
3. 信号质量优先于自动化程度
4. 每个任务结束后，Claude 根据本清单发出下一条工作指令

---

## 阶段一：SOP 完整闭环（当前阶段）

目标：用户每天打开 START_HERE.html，看到当日最新日报，完成决策前判断。

G-01 [ ] SOP自动化串联
      auto_briefing.py 运行后自动更新 START_HERE.html 和 latest_*.md
      当前状态：需要额外手动步骤，SOP存在断点
      验收标准：用户运行一条命令后，START_HERE.html 自动显示当日结论

G-02 [ ] 昨日验证机制
      每次运行 auto_briefing.py 时，自动检查昨日结论是否被市场验证
      验收标准：简报输出中包含"昨日判断：TRANSITION B级 → 实际：[涨跌结果]"

G-03 [ ] 四报表去重
      日报/方向卡/风险卡/执行卡各自只回答一个问题，消除内容重叠
      验收标准：用户30秒内从任意一份报表得到该报表应该回答的答案

## 阶段二：信号质量提升

目标：让 A/B/C 信号有实际胜率记录支撑，而不是规则假设。

G-04 [ ] 信号胜率积累
      trade_record_log_v1.md 开始有实际记录，触发器各等级≥10次后输出胜率
      验收标准：首次月度归因报告生成

G-05 [ ] 周期定位多维化
      在 VIX+美债 基础上，增加 ETF Flow 和美股技术结构作为辅助维度
      验收标准：置信度判断有3个以上独立维度支撑

G-06 [ ] BTC 信号修复
      解决 BTC DATA_GAP 问题（当前长期缺失），使加密信号可用
      验收标准：BTC 信号连续5个交易日返回 OK

## 阶段三：账户数据接入（只读）

目标：自动读取四账户持仓数据，用于报表生成，不接下单接口。

G-07 [ ] user_config.json 规范化
      建立标准格式，用户每日更新加密仓位和本月收益率
      验收标准：auto_briefing.py 能从 user_config.json 读取配置而非 DATA_GAP

G-08 [ ] 持仓快照接入（只读）
      在用户允许的条件下，读取券商导出的持仓 CSV
      原则：只读，不写，不下单，不存储密码
      验收标准：执行卡中四账户显示真实持仓，而非占位符

## 阶段四：40% 目标追踪

目标：建立可量化的目标追踪机制，知道距离40%还差什么。

G-09 [ ] 月度收益追踪
      每月末生成归因报告：实际收益 vs 40% 目标，差距来源归因到具体模块
      验收标准：首次月度追踪报告

G-10 [ ] 策略迭代闭环
      基于归因报告，每季度更新一次策略参数（触发器阈值、仓位系数）
      验收标准：首次策略迭代完成，有前后对比记录

## Claude 工作指令规则

每个任务 TASK_CLOSED 后，Claude 按以下顺序决定下一条工作指令：

1. 查看本清单，找到最高优先级未完成项（[ ] 标记）
2. 判断当前系统状态是否满足该项的前置条件
3. 若满足：发起该项的 NEXT_PROPOSAL
4. 若不满足：发起前置条件的 NEXT_PROPOSAL
5. 每次只发起一个任务，不并行

当前最高优先级：G-01（SOP自动化串联）
