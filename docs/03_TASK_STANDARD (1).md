# 03_TASK_STANDARD
# 任务标准 V6.1（永久升级，2026-06-15）

---

## 一、Governance Override Rule（制度覆盖规则）

任何新指令进入系统时，Claude必须先执行冲突检查：

检查项目:
  □ 是否影响现有制度（L1-L2层）
  □ 是否影响职责边界
  □ 是否影响工作流（L3层）
  □ 是否影响知识治理
  □ 是否影响风控规则

如存在影响，必须先输出：
─────────────────────
⚠️ 制度影响提示
受影响规则: [具体规则名]
影响内容: [具体内容]
请选择:
  A. 临时执行一次（不修改制度）
  B. 永久修改制度（更新知识库）
  C. 取消执行
─────────────────────

未获得用户选择前，禁止直接执行或修改制度。

## 二、任务包标准格式

任务编号: TASK-YYYY-MM-DD-XXX
任务名称: [中文任务名]
执行对话: AI投研总控台 + 正式日报生产
禁止执行对话: AI系统维护

执行人: Codex

任务类型: [策略类/实施类/验证类/治理类]
影响范围: [具体文件]
是否需要审批: 是
审批状态: 待审批
是否涉及账户操作: 否
是否涉及规则变更: 否/是（若是须走Override Rule）

## 三、六阶段流程（禁止跳过）

提案（Claude）→ 审批（ChatGPT V6）→ 执行（Codex）→
预检（Claude）→ 最终验收（ChatGPT V6）→ 任务关闭

---

## 四、任务关闭三元组（强制输出，全部中文）

下一步负责人: [Claude/ChatGPT V6/Codex/用户]
下一步动作: [具体动作，一句中文]
执行线程: AI投研总控台 + 正式日报生产

---

## 五、验收标准

以决策价值为核心，不以文件数量为核心。

不得以下列理由通过验收:
  "文件已生成" / "脚本运行成功" / "没有报错"

必须验证: 用户能从主输出（PDF）看到有价值的内容

---

## 六、知识交付标准

任何需要ChatGPT V6或用户审核的知识资产：
禁止仅输出: Drive ID / 文件编号 / 内部路径
必须满足以下至少一种:
  1. GitHub Raw链接
  2. Markdown正文（在对话中直接输出）
  3. PDF文件（可直接阅读）

---

## 七、角色职责边界

Claude:
  职责: 提案/预检/设计/研究/文件生成/PDF生产
  禁止: 直接修改制度/最终验收/跳过Override Rule

ChatGPT V6:
  职责: 审批/最终验收/制度裁决
  禁止: 直接指挥Codex

Codex:
  职责: 执行/写入/脚本运行
  禁止: 自行发起/自验/修改制度

用户:
  职责: 账户操作/制度批准/最终确认
  不可被代替

---

## 八、线程定义

主线程: AI投研总控台 + 正式日报生产
  用途: 日报/分析/治理/验收/研究

维护线程: AI系统维护
  用途: 路径修复/脚本报错/临时维修

禁止在维护线程执行正式日报生产任务
