## 2.6 生产=实时数据（董事长2026-07-15拍板·2026-07-17强化为明文铁律）

> #### ★★"当日"= 验收当天，机器可核三项（2026-07-17 董事长拍板·强化）
> 交验收前必须**逐项核过**，三项**全部**等于验收当天才算数：
> 1. **五册页头 `data_date` = 验收当天**；
> 2. **`run_id` 时间戳 = 当天**；
> 3. **`production` 快照 = 当天**。

> **旧数据重渲不算生产**：run_id 非当天、或页头横幅标了「N天前的·不是今天」的册，
> **一律不得当验收成品交**——那只能叫**整改自测件**，报告里必须写明是自测件。
> 实时源连不上 → **如实报「连不上·未生产」**，严禁旧数据顶充。
>
> 判定标准（机器可核，不靠自觉）：
> - 产品页头必须有 **data_date（数据日）+ 生产时间 + run_id** 的**超大横幅**，一眼可辨新旧；
> - 五册的 data_date / run_id **必须完全一致**（出厂 lint `L2 同源` 硬闸，不一致即不出品）；
> - `R3 快照单调闸`：本次 production 快照早于上次 → **拒绝生成**，防旧快照顶充；
> - 出厂后由 `product_manifest.py` 登记各册指纹；开工跑 `--check` 比对，
>   对不上 → 报「疑似被旧版覆盖」，**不许拿那份当今天的产品**。
>
> **实时源连不上时的唯一正确动作**：如实报「连不上·未生产」，**不许拿旧数据顶充硬跑**。
> 报「没做到」不扣分，**假报「做到了」是信任击穿**。

## 2.7 每日自动生产（董事长2026-07-17拍板：不该靠他每天提才扫）

**已上线**：Windows 任务计划 `AI投资系统_每日自动生产`，**每天 07:30 JST 自动跑**，无需任何人点。
- 入口：`scripts/daily_auto_produce.py`（13 步：富途实时持仓→持仓真表→审持仓→production→均线→估值→当日涨跌→当日新闻→研报佐证→记分卡→复盘→三件魂→渲五册）
- 出品：当天五册（页头「今天的」·run_id/data_date=当天）+ 自动归档非当天的册到 `00_请先看这里/_历史归档/每日产品/`
- 台账：`data/logs/auto_produce_runs.json`；同时自动回写主控文件的「每日自动生产」块

### 怎么改跑的时间（董事长随时可调）
```
schtasks /Change /TN "AI投资系统_每日自动生产" /ST 08:00     # 改成 08:00
schtasks /Query  /TN "AI投资系统_每日自动生产"               # 看当前设置
schtasks /Run    /TN "AI投资系统_每日自动生产"               # 立刻手动跑一次
schtasks /Delete /TN "AI投资系统_每日自动生产" /F            # 停掉自动
```
或跑 `python scripts/daily_auto_produce.py --install --time 08:00` 重装。
**为什么默认 07:30**：日股开盘（09:00）前、美股昨夜已收 → 董事长早上打开就能看到当天的。

### 失败怎么办（铁律落地）
- **关键环失败**（OpenD 连不上／持仓拉不到／production 挂）→ 立即停、**当天不出品**，台账与主控记「未生产·原因」，**目录里不留旧版冒充今天**。
- **出厂 lint FAIL** → 旧册不被覆盖，记错因。
- 修好后 `schtasks /Run /TN "AI投资系统_每日自动生产"` 补跑即可。
- ⚠ 前提：**OpenD 客户端要开着**，任务计划只在用户已登录时跑。
