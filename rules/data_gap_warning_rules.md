# 数据缺口提示机制规则

更新时间：2026-06-06 14:42:17 +09:00

本规则把系统里的“数据不足 / 旧快照 / C级代理 / 未提供 / 读取失败”统一成：数据缺口 → 缺口等级 → 影响模块 → 禁止动作 → 降级动作 → 补数据路径 → 写回位置 → PDCA复核。

## 绿色：无关键缺口

含义：

* 核心数据已覆盖
* 数据新鲜度满足日报判断
* 不存在影响账户动作的重大缺失

允许动作：

* 可用于日报判断
* 可进入证据链
* 可进入PDCA
* 仍不可自动下单

## 黄色：有缺口，但已降级处理

含义：

* 有数据不足、旧快照、C级代理或部分读取失败
* 已明确标记
* 不触发确定交易动作
* 不影响主要方向判断，或只影响局部模块

允许动作：

* 可作为结构参考
* 可进入PDCA复核
* 可保留观察
* 不得输出确定交易动作

## 红色：关键缺口，阻断动作

含义：

* 缺失会影响日报核心判断
* 缺失会影响账户动作
* 状态文件冲突
* next_task异常
* 账户数据缺失但仍输出动作
* 行情/宏观/资金流/账户数据关键缺失未降级

禁止动作：

* 禁止标绿
* 禁止输出确定账户动作
* 禁止自动推进下一任务
* 禁止把旧快照当当前数据
* 禁止把C级代理当A/B证据

## 通用写回规则

* 绿色缺口状态写回 latest_evidence_chain.md 与 latest_pdca_checklist.md，用于后续复核。
* 黄色缺口必须写明“不触发动作”，并进入明日PDCA。
* 红色缺口必须写入 data_gap_blocking_check.md，并阻断任务标绿或账户确定动作。
* 所有涉及交易的动作仍必须人工核对券商实时行情、账户持仓、现金、成本和挂单。
