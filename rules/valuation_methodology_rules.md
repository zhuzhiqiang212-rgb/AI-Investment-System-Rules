# 估值方法学（尺）· 按类型自动选模型 + PDCA改善依据

来源：右栏_估值方法学.html（Cowork理解岗起草·董事长定 2026-07-14）｜性质：右栏"尺"·慢变·改本尺走董事长｜对应右栏⑤过滤·第3关估值。
GitHub 版本化副本；G盘工作副本见 `00_请先看这里/右栏_估值方法学.html`。落地引擎：`scripts/security_classifier.py` + `scripts/valuation_engine.py`。

## 0. 这块是什么
估值的"方法学尺"：定义**哪类标的用哪个估值模型、什么假设、可信度怎么标**。作用：①让估值可审可改（PDCA对照）；②让估值**按类型自动选模型**，候选池标的变了也自动适配，不靠人一只只指定。

## 1. 类型 → 模型（系统按类型自动选·不手工每只）
系统先按"生意模式"把标的自动归类，再自动套对应模型；新进候选一样自动分类。人只供真财务输入、不指定模型。

| 类型（怎么认） | 用哪个模型 | 真输入 |
|---|---|---|
| **成长股**：盈利稳定增长、EPS≈可分配现金（英伟达/微软/第一三共/多数应用软件） | 两段式 EPS-DCF | forward EPS + 增长/年限/永续/折现（假设明标） |
| **控股/投资公司**：价值＝手里一堆资产（软银） | 净资产法 NAV | 各资产真估值合计 − 净负债 ÷ 股本（±控股折价） |
| **周期股**：盈利大起大落（半导体设备/存储/代工，爱德万/闪迪/台积电） | 中周期盈利法（**不用峰值**） | 正常年景 EPS × 中周期 PE，或 中周期 EBITDA × EV/EBITDA |
| **保险**：利润≠可分配现金（东京海上） | PB / 内含价值法 | 每股净资产 × 目标 PB，或 内含价值/股 × 倍数 |
| **资产**：无财报的资产（BTC/ETH） | 不做企业估值 | 只按仓位纪律(≤12%)＋币价管 |
| **其它**：券商→正常化 PE；综合商社→NAV/PB（按主业归类） | 按主业映射到上面某法 | 对应真输入 |

**铁律**：模型由**类型自动定**，不手工每只指定；硬套错模型会给错价（例：给周期股爱德万套成长DCF，会算出¥1万 vs 现价¥2.9万的荒谬值）。缺真输入的标"待接真源"、不硬编。

## 2. 假设与可信度（怎么标）
- **假设明标可审**：增长率/折现率/中周期年景/控股折价等，逐条写出、标来源；取值偏保守。
- **可信度分级**：`A·精算`（DCF/NAV/中周期·真输入算出）｜`B·相对估值`（现价跟历史/同行比）｜`待接真源`（缺真输入·不硬编）。
- 产品每只卡的"贵不贵/买卖价"要标清用了哪个模型＋可信度，并可点回本尺。

## 3. PDCA 挂钩（为改善做依据）
每只记录一条：用了哪个模型＋关键假设＋当时价 vs 估值区间。复盘时对照估值准不准→若偏差大，判断是"假设该改"还是"选错模型/该换法"→沉淀回本尺。方法学随证据迭代、可回溯（总则第五/十二条），非一拍脑袋定死。

## 6. 总则自检
动静分清（只锚指标定义+门槛尺+方法映射·不锚死名单）｜缺不编（标待接可复核）｜改本尺（增删类型/换模型/改假设口径）＝改尺·走董事长。
