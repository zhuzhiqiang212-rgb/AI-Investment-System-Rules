# 数据缺口与异常处理规则

更新时间：2026-06-05 15:53:11 +0900

1. 缺行情，不得输出确定价格判断。
2. 缺账户，不得输出确定交易动作。
3. 缺情报，不得假设已读取。
4. 旧快照不得进入当前价主表。
5. 异常涨跌幅必须人工复核：±5%人工复核，±10%重大异动。
6. 用户反馈与系统数据冲突时，标记“待人工复核”，不得强行覆盖。
7. 数据不足时，动作降级为观察/不触发动作。
8. 所有缺失和异常必须写入 coverage_score_summary.md 与 PDCA。
9. 部分覆盖不得冒充完整覆盖；框架存在不得等同于定量数据存在。
10. 账户截图和账户快照只作结构参考，所有交易需人工确认。

# 覆盖率红色阻断规则

如果以下任一模块为红色：

- 美股核心行情
- 日股核心行情
- 宏观行情
- 情报输入
- 账户输入

则：

1. P1-014 不能标绿。
2. 日报必须显示“数据覆盖不足”。
3. 对缺失标的不得输出当前价格判断。
4. 对缺失宏观变量不得触发账户动作。
5. 对缺失情报不得假设已读取。
6. 四账户动作必须降级为“结构参考 / 待人工复核”。
7. dashboard 必须显示红色阻断原因。

当前红色模块：无（宏观缺口已降级处理；P1-014仍待ChatGPT验收）。



# 宏观缺失项降级规则

如果宏观变量没有稳定公开定量源：

1. 标记为“背景/事件状态”。
2. 不触发账户动作。
3. 不参与当前红黄绿灯号计算。
4. 进入PDCA明日复核。
5. 写入coverage_score_summary。
6. dashboard显示为“已降级处理”。
7. 不得编造数值。
8. 不得用替代错误指标冒充。
