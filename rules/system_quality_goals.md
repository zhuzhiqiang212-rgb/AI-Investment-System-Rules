# AI投研系统总质量目标

## 1. 系统最终目标

系统不是为了生成一堆文件。

系统最终目标是帮助用户每天判断：

1. 今天市场结构有没有变
2. 今天该进攻、观望，还是防守
3. 四个账户分别该看什么
4. 哪些标的可以等回踩
5. 哪些标的不许追
6. 哪些风险正在升温
7. 哪些数据缺口会影响判断
8. 明天要验证什么
9. 什么条件下判断失效
10. 所有实际交易前必须人工核对券商实时行情

## 2. 用户版日报质量目标

用户版日报必须达到：

- 5秒内看懂今天是进攻 / 观望 / 防守
- 30秒内看懂四账户动作
- 1分钟内看懂最大风险和禁止动作
- 每个重要结论必须有：结论、证据、影响账户、当前动作、等待条件、失效条件
- 不允许只有口号式结论
- 不允许只写“观察”
- 不允许没有证据的动作建议
- 不允许把数据不足伪装成结论
- 不允许所有模块都写“数据不足”逃避判断

## 3. 短线价格指导质量目标

短线价格指导必须分两类：

### A. 可给参考价区间

如果具备以下至少一种依据，就必须给参考价区间：

- 公开延迟行情
- 前低 / 近期支撑
- 20日 / 50日 / 200日均线
- 近期高低点区间
- 用户提供的价格观点
- 技术支撑 / 压力位
- 波动区间
- 账户已有成本 / 持仓信息

参考价必须标注：行情口径、不可直接下单、操作前人工核对券商实时行情、适合已有仓还是新仓、小仓/观察仓/禁止新增、失效条件。

### B. 不可给参考价

只有在完全缺少可复核价格、技术区间、持仓成本、用户价格观点时，才允许写：数据不足，暂不输出价格。

但必须说明：缺什么数据、用户该补什么、补到哪里、补齐后如何生成价格卡。

## 4. 四账户动作卡质量目标

每个账户必须写清楚：当前暴露、当前风险、今天动作、禁止动作、重点观察标的、如果风险上升怎么办、如果风险缓和怎么办、人工确认项。

禁止只写：观察、控风险、不追高、等承接。这些词可以出现，但必须配原因和条件。

## 5. 模糊表达禁止规则

以下表达不能单独作为最终输出：AI未坏但分化、高Beta风险复核、不误杀、等承接、只作观察、资金流不影响动作、数据不足、宏观偏热、估值偏高。

如果使用这些表达，必须同时写：具体指什么、看哪个证据、影响哪个账户、今天怎么做、什么条件下改变判断。

## 6. 收口门槛

如果用户版日报存在以下任一问题，不得最终收口：

- 第一屏没有明确进攻 / 观望 / 防守
- 四账户动作不够具体
- 短线价格模块没有给出可用参考价，也没有说明为什么不能给价
- 重要结论缺少证据
- 重要动作缺少失效条件
- 大量模块只是规则摘要，不是今日判断
- 后台验收内容挤占用户主阅读
- 用户每天还需要自己判断哪个文件有用
- 系统需要用户人工检查质量才知道不达标
