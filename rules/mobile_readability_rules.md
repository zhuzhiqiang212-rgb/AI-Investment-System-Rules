# 手机端可读性规则

更新时间：2026-06-05 23:52:39 +09:00

## 1. 适用页面

- START_HERE.html
- readable_dashboard.html
- latest_official_daily_report_panel.html
- MASTER_TASK_DASHBOARD.html
- EVIDENCE_CHAIN_DASHBOARD.html

## 2. 固定标准

1. 页面必须声明 `meta viewport`，正文基础字号不得低于 18px。
2. 手机端宽度不超过 760px 时，主网格必须自动切换为单列。
3. 页面不得依赖左右拖动；正文、链接、按钮和长字符串必须自动换行。
4. 第一屏必须显示：今日一句话、系统状态、下一步唯一任务、账户动作摘要、日报入口和交易边界。
5. 第一屏不得放横向长表；详细证据、基本面、估值和任务明细放在后续卡片或折叠区。
6. 长表格在手机端必须转换为块状卡片；不得用黑底小字展示用户阅读内容。
7. 同一任务不得出现新旧状态冲突；页面状态以 master_task_status.json 为准。
8. 用户入口必须提供明显按钮，不要求用户识别路径或打开原始 Markdown 长表。
9. 交易边界必须醒目：延迟行情可用于日报判断，不可直接交易；交易前人工核对券商实时行情、账户、现金、成本和挂单。
10. 每次页面更新后，至少用 390×844 手机视口做一次真实渲染检查。


## 3. 主要模块卡片规则

- 每张卡必须有结论、证据、风险、动作、失效条件。
- 不允许只有大段文字，不允许只有横向大表。
- 长表只能作为详情；手机入口必须先显示卡片摘要，字段过多时必须拆成卡片。

## 4. 字体与视觉规则

- 禁止黑底小字。
- 禁止灰底低对比度。
- 禁止横向挤爆。
- 重要结论必须醒目。

## 5. 导航规则

- START_HERE 是第一入口。
- readable_dashboard 是第二入口。
- latest_official_daily_report_panel 是正式日报入口。
- 历史文件不能作为今日主入口。

## 6. 验收规则

- 如果任一核心入口 D 级，P1-022 不能标绿。
- 如果下一步任务找不到，P1-022 不能标绿。
- 如果状态冲突，P1-022 不能标绿。
- 如果主要动作卡不可读，P1-022 不能标绿。
## 7. 可读性等级

- A：第一屏可在 5 秒内找到核心动作，移动端无横向溢出。
- B：核心入口清楚，但完整内容需向下滚动。
- C：主要内容需要多次查找，或局部信息密度偏高。
- D：关键入口不可见、横向挤爆、黑底小字或状态冲突，必须阻断验收。

## 8. P1-022 边界

P1-022 当前保持黄色，等待 ChatGPT 验收；Codex 不得自行标绿，也不得推进下一任务。

# 界面可读性同步优化原则

更新时间：2026-06-06 13:27:47 +09:00

不要等系统全部完成后才优化页面。  
从现在开始，每新增一个模块，必须同步满足最低可读性标准。

## 最低可读性标准

1. Markdown 长表只能作为后台明细，不得作为用户主入口。
2. 用户主入口必须优先使用 HTML 卡片版。
3. START_HERE、readable_dashboard、latest_official_daily_report_panel 必须优先展示卡片摘要。
4. 每个模块第一屏必须能看到：一句话结论、当前状态、当前动作、禁止动作、等待条件、失效条件。
5. 长表必须折叠到“查看明细”区域。
6. 手机端必须单列显示。
7. 字体不得太小，正文建议不低于18px。
8. 禁止黑底小字作为主阅读页面。
9. 如果某个模块只有 Markdown 长表，没有 HTML 卡片版，则该模块不能算完成。
10. 最终全部任务完成后，再执行一次统一 UI 美化任务。

## 当前策略

- 现在：同步做最低可读性，保证每天可用。
- 最后：做一次全局统一美化，不影响内容逻辑。
