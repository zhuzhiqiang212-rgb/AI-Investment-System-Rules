# 用户版日报宪法层 V1

## 0. 宪法目标

用户版日报不是系统验收包，不是模块目录，不是压缩摘要。

用户版日报的唯一目标是：用户每天打开一个入口后，能够按层级完成决策：

1. 先知道今天和昨天相比发生了什么变化。
2. 再知道今天该不该动。
3. 再知道四个账户分别怎么处理。
4. 再知道最大风险、关键证据和失效条件。
5. 需要时可以展开完整 Direction Card、Risk Card、Execution Card、Evidence Chain、PDCA 和四账户动作卡。

## 1. 永久规则与可调整规则

### 永久规则

以下规则不可删除，不可因美观、简洁或速度而牺牲：

- 今日 Delta 卡必须存在。
- 四账户唯一动作必须存在。
- 决策证据链五段结构必须存在：结论、证据、影响、动作、失效条件。
- 失效条件必须存在。
- 第一屏编辑层必须存在。
- Evidence Chain 必须保留并可展开。
- PDCA 必须保留并可展开或可导航。
- Direction Card / Risk Card / Execution Card 必须保留完整内容。
- 禁止自动下单。
- 禁止把延迟行情当实时交易价。
- 实际交易前必须人工确认券商界面价格、持仓、现金、成本、挂单。
- 用户满意前不得最终收口。

### 未来可调整规则

以下可以随用户偏好和系统成熟度调整：

- 卡片颜色与视觉风格。
- 第一屏模块排序，但 Delta、账户动作、最大风险不得下沉到不可见位置。
- 展开模块默认打开或折叠。
- 评分权重。
- 页面中使用 Markdown 原文展示还是转换为更精细 HTML 卡片。
- 日报文件名和入口文件名，只要唯一入口不混乱。

## 2. 今日 Delta 卡

### 目的

回答用户最重要的问题：今天和昨天相比发生了什么变化。

### 标准

必须包含：昨日判断、今日判断、变化、证据、影响、动作、失效条件。没有变化也必须写“延续”。

### 验收条件

- 用户能快速找到 Delta 卡。
- Delta 卡必须说明变化影响哪个账户和哪些标的。
- Delta 卡必须说明今天动不动。
- Delta 卡必须说明什么条件下今天的变化判断失效。

### 失败条件

- 只写“市场变化”“风险增加”等模糊话。
- 没有证据。
- 没有账户影响。
- 没有失效条件。
- Delta 被埋在后台或长表后面。

## 3. 四账户唯一动作

### 目的

回答用户：富途、IBKR、SBI个人、bitFlyer 今天分别怎么处理。

### 标准

每个账户必须有一个顶部唯一动作，格式为：不新增 / 持有观察 / 风险复核 / 等承接 / 降风险观察 / 只作温度计，并且必须落到标的和条件。

### 验收条件

- 四账户全部出现。
- 每个账户有唯一动作。
- 每个账户有重点标的。
- 每个账户有禁止动作。
- 每个账户有人工确认项。

### 失败条件

- 只写“观察”“控风险”“不追高”。
- 没有标的。
- 没有条件。
- 没有人工确认项。

## 4. 决策证据链

### 目的

防止日报只有口号，没有可追溯判断。

### 标准

所有重要结论必须使用五段结构：

- 结论
- 证据
- 影响
- 动作
- 失效条件

### 验收条件

- 今日总判断满足五段结构。
- Delta 卡满足五段结构。
- Direction Card / Risk Card / Execution Card 的用户可见摘要满足五段结构。
- 四账户动作满足五段结构或等价结构。

### 失败条件

- 缺证据。
- 缺失效条件。
- 动作是通用原则而非今日动作。
- 影响没有落到账户/标的/风险。

## 5. 失效条件

### 目的

让用户知道当前判断什么时候作废。

### 标准

每个关键判断必须有触发重评的具体条件。失效条件可以是价格/趋势/账户/宏观/风险温度/资料证据变化。

### 验收条件

- 用户能快速找到今日主要失效条件。
- 每个账户动作有风险升级或降级条件。
- PDCA 能承接失效条件。

### 失败条件

- 写“继续观察”但不说明看什么。
- 失效条件没有触发方向。
- 失效条件无法写回 PDCA。

## 6. 第一屏编辑层

### 目的

不是为了变短，而是为了让完整日报有阅读顺序。

### 标准

第一屏编辑层必须包含：

- 今日 Delta 主卡。
- 四账户唯一动作总览。
- 今日最大风险。
- 今日最关键证据。
- 今日失效条件导航。
- 详细模块导航。

### 验收条件

- 用户打开后先看到“今天有什么变化”。
- 用户能快速找到“今天该不该动”。
- 用户能快速找到“四账户怎么处理”。
- 用户能快速跳转到完整证据。

### 失败条件

- 第一屏是系统验收语言。
- 第一屏只有路径和 YES/NO。
- 第一屏只有规则摘要。
- 第一屏没有账户动作。

## 7. Evidence Chain

### 目的

保留完整证据，防止为了可读性牺牲决策质量。

### 标准

Evidence Chain 必须作为完整展开层存在，不得被摘要替代。

### 验收条件

- 页面有 Evidence Chain 入口。
- Evidence Chain 可展开或可打开。
- 关键结论能追溯到证据。

### 失败条件

- 删除 Evidence Chain。
- 只保留摘要，没有完整证据。
- 用户无法从结论跳到证据。

## 8. PDCA

### 目的

把今天判断转化为明天验证和写回机制。

### 标准

PDCA 必须包含：今日判断、明日验证指标、如果正确、如果错误、动作调整、失效条件、写回位置。

### 验收条件

- 页面能找到明日 PDCA。
- 今日失效条件能进入 PDCA。
- 四账户动作变化能写回。

### 失败条件

- 只有“继续观察”。
- 没有写回位置。
- 明日验证不具体。

## 9. Quality Gate 评分机制

总分 100。

建议权重：

| 项目 | 分值 |
|---|---:|
| Delta 卡 | 15 |
| 四账户唯一动作 | 15 |
| 决策证据链五段结构 | 15 |
| 失效条件 | 10 |
| 第一屏编辑层 | 15 |
| Evidence Chain 完整保留 | 10 |
| Direction/Risk/Execution 完整保留 | 10 |
| PDCA 可执行 | 10 |

### 进入 dashboard 规则

- 90分及以上：允许进入 dashboard。
- 80-89分：只能作为待修复预览，不得作为最终用户入口。
- 80分以下：禁止进入 dashboard。

### 硬阻断

出现以下任一项，不论总分多少，都禁止进入 dashboard：

- 缺 Evidence Chain。
- 缺四账户动作。
- 缺失效条件。
- 缺 Delta 卡。
- 只显示系统验收语言。
- 自动下单或暗示可直接交易。
- 把延迟行情当实时交易价。
- 修改账户数据或投资结论但未说明来源。

## 10. 与 Worker 治理关系

- Codex 可以执行生成和更新，但不能最终验收。
- Claude 可以做内容质量预检，但不能写回 Drive 和不能标绿。
- Cursor 可以做 dashboard 工程实现，但不能修改投资结论、账户数据或最终验收。
- ChatGPT / AI投研总控台 V4 是最终验收人。
