## 二、账户目标

### 长期目标

建立一个围绕 AI超级周期、全球流动性、AI Capex/ROI、数字资产、日股AI链、A股AI国产替代的多账户投研系统。

核心不是每天预测涨跌，而是：

1. 判断市场是否已经变了
2. 判断当前该进攻、观望还是防守
3. 提高风险收益比
4. 避免追高
5. 避免恐慌卖出
6. 避免高Beta失控
7. 让账户动作有证据、有条件、有复盘

### 账户核心原则

- 核心AI仓不轻易误杀
- 高Beta仓不追涨
- 日股AI链强势时不追，急跌时等承接
- 加仓等市场悲观、dip buy耗尽、承接确认
- 减仓等情绪修复、反弹、催化出现
- 所有交易前必须人工确认账户、持仓、成本、现金、挂单

## 三、投研目的

本系统不是写文章。
本系统不是堆新闻。
本系统不是只看行情涨跌。

本系统要建立：

结论 → 证据 → 估值 → 基本面 → 资金流 → 仓位 → 风险 → 动作 → 条件 → 失效 → PDCA

的完整逻辑证据体系。

每天必须回答：

1. 今天能不能动？
2. 已有仓怎么办？
3. 新仓怎么办？
4. 能不能加仓？
5. 能不能减仓？
6. 等什么？
7. 哪些证据支持？
8. 哪些证据反对？
9. 错了怎么办？
10. 明天怎么验证？

## 四、证据链结构

每个重要结论都必须包含：

| 项目 | 要求 |
|---|---|
| 结论 | 一句话说明 |
| 支撑证据 | 行情、资料、宏观、资金流、基本面 |
| 反向证据 | 哪些因素可能推翻判断 |
| 估值/拥挤度 | 是否偏贵、是否拥挤 |
| 基本面质量 | FCF、资产负债表、毛利率 |
| 账户影响 | 影响富途、IBKR、SBI、bitFlyer哪个账户 |
| 当前动作 | 做什么 |
| 禁止动作 | 不做什么 |
| 等待条件 | 等什么信号 |
| 失效条件 | 什么情况判断作废 |
| PDCA验证 | 明天/下一次怎么验证 |

禁止只写：

- 观察
- 等待
- 趋势强
- 过热
- 风险高

必须写清楚：观察什么、等什么、什么条件能动、什么条件不能动。

## 五、颜色状态规则

所有日报和 START_HERE 必须使用颜色状态：

### 绿色

含义：主线健康，可以持有观察。

注意：绿色不等于可以追高。

### 黄色

含义：趋势还在，但风险收益比一般，适合等待确认。

动作：已有仓观察，新仓等回踩。

### 红色

含义：风险升高，停止新增，进入复核或防守。

动作：不追、不加、高Beta先控风险。

### 灰色

含义：数据不足，不能下结论。

动作：等待数据，不能生成确定性动作。

## 六、强势 / 过热 / 急跌 / 破趋势定义

- 强势：趋势仍在，已有仓可观察，不等于新仓可追。
- 过热：不是看空，而是追高性价比下降。
- 急跌：不是自动买点，而是承接验证。
- 破趋势：不是正常回踩，而是风险升级。
- 缩量回踩：可能是健康回调，等待承接。
- 放量下破：风险升级，进入减风险复核。

## 七、行情数据规则

日报必须明确显示：

- 数据来源：Yahoo chart public API / public market data / 其他实际来源
- 是否交易所实时：YES/NO
- 是否券商实时：YES/NO
- 是否可用于直接下单：YES/NO

如果不是交易所实时行情或券商实时行情，必须写：

公开行情快照不用于直接下单。所有盘中交易必须人工核对券商实时行情后执行。

如果没有实时行情，禁止输出：

- 现在买
- 现在卖
- 立即加仓
- 立即减仓

只能输出：

- 条件式观察
- 等待确认
- 小仓试探条件
- 人工复核后执行

## 八、公司基本面质量规则

所有重要公司必须长期检查：

1. 自由现金流 Free Cash Flow
2. 健康资产负债表 Balance Sheet Health
3. 高毛利率 Gross Margin

解释：

- 自由现金流强，说明公司不是只靠故事，而是真能产生现金。
- 资产负债表健康，说明公司现金、债务、融资压力可控。
- 高毛利率说明公司有定价权、技术壁垒或平台优势。

必须固定输出：

| 标的 | FCF质量 | 资产负债表健康度 | 毛利率质量 | Capex压力 | 基本面等级 |
|---|---|---|---|---|---|

禁止编造财报数字。如果没有最新数据，必须写“数据不足”。

## 九、估值与性价比规则

必须保留估值与性价比表。

字段：

| 标的 | 所属方向 | 估值状态 | 性价比等级 | 风险等级 | 当前动作 | 等待条件 |
|---|---|---|---|---|---|---|

性价比等级：

- S：核心优先，回调值得关注
- A：强趋势，等回踩
- B：高Beta或估值偏高，控制仓位
- C：数据不足或风险较高
- D：暂不参与

规则：

- 强势不等于可以买。
- 便宜不等于可以买。
- 急跌不等于机会。
- 必须同时看价格、估值、基本面、资金流、账户暴露、宏观。

## 十、账户执行规则

### 富途

重点：美股核心AI + 高Beta + 部分日股。

必须关注：

- MSTR / COIN 高Beta风险
- MSFT / NVDA / TSM 核心AI
- 9984 日股AI映射

规则：不追高Beta，不误杀核心AI。

### IBKR

重点：现金缓冲较弱，高Beta暴露需要优先复核。

规则：IBKR如果现金低，不能扩大COIN/MSTR风险。

### SBI个人

重点：9984 / 6857 日股AI链集中。

规则：强势持有观察，急跌等承接，不自动加仓。

### bitFlyer

重点：BTC / ETH 只做风险温度计。

规则：不新增加密现货，先观察BTC/ETH是否止跌。

## 十一、已有仓 / 新仓分开规则

任何标的必须区分：

- 已有仓：持有 / 观察 / 减风险复核 / 不恐慌卖
- 新仓：不追 / 等回踩 / 等止跌 / 小仓试探 / 暂不参与

禁止只写：“持有观察，不追高”。

必须写：已有仓怎么做，新仓怎么做。

## 十二、软银9984 / 爱德万6857特殊规则

如果 9984 或 6857 大涨：

- 状态：强趋势上涨。
- 动作：已有仓持有观察，新仓不追，等回踩。

如果 9984 或 6857 急跌，例如 -5% 到 -10% 或更大：

- 状态：强趋势急跌验证中。
- 动作：不是自动买点；不立刻重仓加仓；不恐慌卖出；等待承接确认。

可以小仓分批的条件：

1. 分时止跌
2. 成交量不再放大砸盘
3. 重新站回VWAP或关键短线均线
4. 收盘不破关键支撑
5. 日元/日本利率没有继续反杀

至少满足3条，才允许小仓分批。

不能接的条件：

1. 放量跌破趋势
2. 反弹无量
3. 日元快速走强
4. 日本利率继续上行
5. AI链同步退潮

## 十三、验资料流程规则

如果用户直接上传资料给 ChatGPT：

1. ChatGPT先判断资料价值。
2. ChatGPT提取重点。
3. ChatGPT给Codex写入指令。
4. Codex只负责写入系统和生成文件。

如果资料放在 Drive，只扫描：

AI_Investment_System/inbox

不要全盘扫描 Drive。

处理完成后移动到：

AI_Investment_System/processed/YYYY-MM-DD

并更新：

AI_Investment_System/system/processed_manifest.json

禁止重复处理已进入 manifest 的文件。

如果 PDF 是图片型：

Codex不要反复OCR失败。标记为图片型PDF。由ChatGPT视觉判断或用户提供文字。

如果是 .gdoc：

不能当作本地正文文件。必须导出为 PDF / TXT / DOCX，或走Google Drive云端读取。

## 十六、Codex与ChatGPT分工

Codex负责：

1. 读取本地文件
2. 生成文件
3. 更新HTML
4. 更新START_HERE
5. 移动归档
6. 更新manifest
7. 执行固定格式任务

ChatGPT负责：

1. 判断资料价值
2. 判断逻辑质量
3. 检查证据链
4. 验收日报
5. 提供修正指令
6. 判断是否通过

Codex不能自说自话把资料直接变成交易结论。ChatGPT验收前，重要日报只能算候选版。

## 十七、PDCA规则

每日必须保留：

| 今天判断 | 明天看什么 | 如果对 | 如果错 | 下一步动作 |
|---|---|---|---|---|

至少包含：

1. AI主线是否继续承接
2. 软银/6857急跌后是否承接
3. COIN/MSTR是否继续弱于BTC
4. CPI前利率压力是否继续
5. dip buyer是否进场
6. 情绪是否从乐观转悲观
7. 账户风险是否降低
8. 新增资料是否被市场验证

## 十八、禁止事项

禁止：

1. 删除完整证据链。
2. 删除估值表。
3. 删除公司基本面质量表。
4. 删除PDCA。
5. 删除账户动作卡。
6. 把公开行情快照当作实时行情。
7. 把急跌自动写成买点。
8. 把过热写成看空。
9. 不区分已有仓和新仓。
10. 编造财报数据。
11. 只输出长文，不输出决策卡。
12. 只输出卡片，删掉证据库。
13. 让用户每次指出缺什么再补什么。
