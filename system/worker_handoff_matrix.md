# Worker Handoff Matrix V1

## 1. 默认角色

| Worker | 默认角色 | 默认交付物 | 默认发给谁 | 最终验收人 |
|---|---|---|---|---|
| Claude | 预检 / 长文分析 / 内容可读性检查 | 预检结论或结构化内容摘要 | 用户 / ChatGPT / AI投研总控台 V4 | ChatGPT / AI投研总控台 V4 |
| Codex | 执行 / 文件生成 / 系统任务执行 / 验收包生成 | 原始验收包、规则文件、系统文件 | 用户上传给 ChatGPT / AI投研总控台 V4 | ChatGPT / AI投研总控台 V4 |
| Cursor | 工程执行 / OCR / PDF处理 / 脚本 / Dashboard / GitHub工程 | 工程产物、OCR结果、脚本、工程验收包 | Codex 汇总或用户上传给 ChatGPT / AI投研总控台 V4 | ChatGPT / AI投研总控台 V4 |
| ChatGPT / AI投研总控台 V4 | 最终验收 | 最终验收结论、通过/返工指令 | 用户 / 原执行Worker | ChatGPT / AI投研总控台 V4 |

## 2. Claude 完成后发给谁

| Claude任务类型 | 输出 | 发给谁 | 下一步 |
|---|---|---|---|
| 长PDF/长PPT内容分析 | 结构化观点、证据、风险点 | Codex 或用户 | Codex 写入系统或生成验收包 |
| 湖水/老雷/TXT/视频转文字分析 | 内容摘要、证据链预检、风险点 | Codex 或用户 | Codex 整合到报告/验收包 |
| 验收包预检 | PASS / FAIL 预检结果 | 用户 | PASS后用户上传原始验收包给 ChatGPT；FAIL则返回Codex/Cursor返工 |
| 可读性检查 | 问题清单、修正建议 | Codex | Codex 修改用户版报表 |

Claude 禁止：写回Drive、修改系统文件、最终验收、Q0任务标绿、修改账户数据、修改投资结论。

## 3. Codex 完成后发给谁

| Codex任务类型 | 输出 | 发给谁 | 下一步 |
|---|---|---|---|
| 规则/系统文件生成 | 规则文件 + 验收包 | 用户上传给 ChatGPT / AI投研总控台 V4 | ChatGPT最终验收 |
| 日报/报告/卡片生成 | 用户报表 + 验收包 | 用户上传验收包给 ChatGPT | 通过后可写回状态 |
| 状态写回 | 状态文件 + 仪表盘更新 | 用户 / ChatGPT复核 | 确认下一步任务 |
| 工程配合任务 | 脚本/说明/验收包 | Cursor或用户 | Cursor执行工程任务或ChatGPT验收 |

Codex 禁止：最终验收、自动下单、未经确认修改账户数据、未经确认修改投资结论、自行标绿Q0任务、用预检代替最终验收。

## 4. Cursor 完成后发给谁

| Cursor任务类型 | 输出 | 发给谁 | 下一步 |
|---|---|---|---|
| OCR处理 | OCR文本、结构化CSV/JSON、OCR验收包 | Codex | Codex整合并生成系统验收包 |
| PDF/PPT解析 | 提取文本、结构化索引、解析报告 | Codex / Claude | Claude做内容分析或Codex写入系统 |
| Dashboard/脚本开发 | 代码、运行说明、工程验收包 | Codex / 用户 | Codex更新入口或ChatGPT验收 |
| GitHub工程维护 | git状态、结构、排除清单、工程验收包 | 用户 / ChatGPT | 用户决定是否进入远程推送 |

Cursor 禁止：修改账户数据、修改投资结论、最终验收、Q0任务标绿、自动下单、修改方向卡/风险卡结论。

## 5. 统一完成判定

任何 Worker 完成后都必须确认：

| 检查项 | 必须结果 |
|---|---|
| 验收包路径 | 有完整路径 |
| 文件是否存在 | YES |
| 文件大小 | 大于0 |
| 最终验收人 | ChatGPT / AI投研总控台 V4 |
| 发给谁 | 明确 |
| 下一步 | 明确 |
| 唯一动作 | 只有一个 |
| 可直接执行指令 | 有 |

缺任一项，任务未完成。

## 6. 标准流转图

```text
Claude / Codex / Cursor 完成任务
↓
生成或确认原始验收包
↓
输出文件存在性、大小、修改时间
↓
用户或指定Worker预检
↓
ChatGPT / AI投研总控台 V4 最终验收
↓
通过后 Codex 才能写回绿色
↓
不通过则返回原执行Worker返工
```
